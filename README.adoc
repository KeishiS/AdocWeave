= AdocWeave

AdocWeaveはRustで実装するAsciiDoc処理系です。損失のないsyntax tree、出力形式に依存しない
semantic AST、診断、整形、HTML変換およびLanguage Serverを提供します。

== 開発

Nix開発環境にはRustツールチェーン、`cargo-make`、`ripgrep`が含まれます。

[source,console]
----
nix develop
----

利用可能なタスクは次のとおりです。

- `cargo make fmt`または`cargo make format`: Rustソースを整形する。
- `cargo make fmt-check`または`cargo make format-check`: ファイルを変更せず整形状態を検査する。
- `cargo make check`: 全target・全featureを型検査する。
- `cargo make clippy`: warningを許可せずClippyを実行する。
- `cargo make test`: workspace全体のテストを実行する。
- `cargo make test-core`: コアとCLIのテストを実行する。
- `cargo make test-lsp`: Language Serverのテストを実行する。
- `cargo make test-grammar`: 規範文法と置換契約のテストを実行する。
- `cargo make test-properties`: syntax範囲、lossless性、Formatter冪等性の不変条件を検証する。
- `cargo make test-release`: CLIとLanguage Serverで共通の初期リリースfixtureを検証する。
- `cargo make check-zed`: Zed拡張crateを型検査する。
- `cargo make clippy-zed`: Zed拡張crateへClippyを実行する。
- `cargo make doc-check`: warningを許可せずAPIドキュメントを生成する。
- `cargo make docs-check`: `.md`ファイルの混入を検出する。
- `cargo make build`: 全target・全featureをビルドする。
- `cargo make run-lsp`: 標準入出力でLanguage Serverを起動する。
- `cargo make verify`、`cargo make ci`または`cargo make`: 変更を伴わない全検証を実行する。
