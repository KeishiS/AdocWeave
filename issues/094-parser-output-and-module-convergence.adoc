= Parser出力型とmodule境界の最終収束
:status: completed
:depends-on: 091,092,093

== 目的

再帰block parserの出力をcontextごとの利用可能なfactへ型付けし、状態と不変条件の整理後に`parser.rs`の責務を
機能単位へ分割する。公開前に一時的な互換wrapperと暗黙に破棄されるfactを残さない。

== 作業

* rootとnestedで共通するblock／syntax／anchor factと、rootだけのheader／document attribute factを分離する。
* `BlockSequenceOutput`をcontext別の型、または共通出力とroot集約器へ置換し、未使用fieldの黙示破棄をなくす。
* parser状態機械、delimiter block、list継続、document headerを責務単位のmoduleへ移す。
* 型を別moduleへ公開するだけの薄いwrapperや循環依存を作らず、共有budget、cancel、source rangeの所有者を明記する。
* raw parse factからlowering、identifier catalog、structure、diagnostic確定までの順序を型と文書で固定する。
* native、WASM、CLI、LSP、Zedおよび全契約fixtureを最終APIへ一斉移行する。
* architecture、roadmap、Issue 067／080／089の完了記録と実装上の差異を明記する。

== 完了条件

nested parserがroot専用factを生成・破棄せず、各moduleの入力と出力に所有する不変条件が記述されている。
`parser.rs`がblock種別固有の全実装を抱えず、新構文の追加箇所がrecognition、semantic construction、consumer契約として
追跡可能である。`cargo make release-gate`が最終構成で成功し、Issue 082以降の公開作業へ進める。

== 実装結果

`BlockSequenceOutput`を`RootBlockSequenceOutput`とnested `BlockFacts`へ分離した。document header、文書属性および
属性問題はroot stateが存在する場合だけ生成され、root／nestedの取り違えは内部不変条件違反として失敗する。
compound blockとAsciiDoc table cellのnested syntax factは親delimiter nodeへ接続し、黙示的な破棄を解消した。

再帰入力・context・cursor・出力不変条件を`block_sequence.rs`、delimiter registryと閉じ位置回復を
`delimiter.rs`、root header状態を`document_header.rs`、list markerと木構築を`list_parser.rs`へ分離した。
list continuation自体は全block種別へ再帰するsequence dispatchであり、専用moduleへ移すとparserへの循環した
callback APIが必要になるためorchestratorへ残した。この境界は行数削減ではなく、共有budget、cancelおよびrangeを
一つの状態機械が所有するという不変条件を優先した結果である。

raw fact、lowering、属性確定、identifier、structure、syntax diagnostic、catalog、lintの順序を
architectureへ規範化した。公開semantic APIは変更せず、nested syntax treeの階層化をCore Profile 17として
native、CLI、LSP、WASMおよびZedが共有する単一`Analysis`へ反映した。

== 完了後レビュー

最初のmodule分割ではdomain moduleが型の再公開元である`parser` facadeを参照し、概念上の循環依存が残った。
完了後レビューでsemantic型の参照先を`block_model`へ変更し、共通実行状態と失敗型を`parser_support`へ分離した。
これにより依存方向はmodel／supportからdomain policy、最後に`parser` orchestratorとなり、新しいdomain moduleが
巨大なfacadeへ逆依存する必要をなくした。
