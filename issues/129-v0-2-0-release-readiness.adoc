= v0.2.0 release readinessと契約凍結
:status: completed
:depends-on: 128

== 目的

`v0.1.0`以後にmainへ統合した標準AsciiDoc互換性拡張と公開成果物の差分を監査し、
`v0.2.0-rc.N`から`v0.2.0` stableまでを同一のrelease trainとして受け入れる。

== 判断

source block短縮記法、list表示属性、bibliography section、TOCおよびsection番号は利用者可視の
機能追加である。`0.y.z`期間の方針に従い、次版はpatchではなくminorの`0.2.0`とする。

公開HTML、projection、conformance、WASM応答または公開Rust APIに互換性のない変更が含まれる場合は、
`CONTRACT_VERSION`を一度だけ更新し、manifest、fixture、互換性文書およびmigration noteを同じ変更で同期する。

== 作業

* `v0.1.0`からの利用者可視差分と契約version更新の要否を監査する。
* workspace、Zed、browser、release manifestおよび配布設定を`0.2.0-rc.1`へ同期する。
* mainのcandidate成功後に`v0.2.0-rc.1`を公開し、9 asset、checksum、attestation、CLI、LSP、browser、ZedおよびNixを受け入れる。
* RC公開後は機能追加を停止し、repository所有者が公開前確認を完了した時点でstableを公開する。
* blocker修正が公開契約を変更する場合は新しいRCを公開する。stableは最終RCと同じ機能baselineに、stable version metadataだけを同期して公開する。

== 検証

* `cargo make release-gate`
* `node tools/release-contract.mjs --tag=v0.2.0-rc.1`
* tag workflowの必須job、公開assetの完全性検証およびclean環境の導入E2E
* Linux x86-64／ARM64、browser Worker、NixOSおよびZedの受け入れ記録

== 受け入れ結果

`v0.2.0-rc.1`は`059dbafa43949806e10c966ca7bfbbb90edd6664`、stable `v0.2.0`は
`c38a274c6c724def56d7e92545c0777ba3813013`をそれぞれtagした。RC Releaseはprerelease、stable Releaseは
非draft・非prereleaseであり、いずれも計画どおり9 assetを含む。

stableのhttps://github.com/KeishiS/AdocWeave/actions/runs/30002513033[Release run 30002513033]は、
quality、Linux x86-64／ARM64 native build・smoke、browser／Zed artifact verification、candidate metadata、
Linux 2 architectureのNix packageを含むinstallation E2Eおよびpublishをすべて成功した。
公開asset集合はCLI／LSP各2 architecture、browser、Zed、manifest、SPDX SBOM、checksumの9 fileであり、
0 byte assetはなかった。`node tools/release-contract.mjs --tag=v0.2.0`も成功した。

`v0.1.0`からの機能差分はsource block短縮記法、list表示属性、bibliography section、TOCおよびsection番号であり、
公開契約versionは2へ更新した。移行手順はxref:../docs/upgrade-0-2.adoc[v0.2.0への移行]へ記録した。
