= ノートアプリ互換プロファイルのリリース検証

== 目的

ノートアプリ互換プロファイルをサーバ、ブラウザ、CLIおよびLanguage Serverで一貫して公開できる状態にする。

== 実装範囲

- 必須構文、未対応構文、厳格・寛容モードおよび既定上限をバージョン付きプロファイルとして固定する。
- 構文プロファイル、公開API、診断・エラー、HTML契約、WASM APIおよびcrateのSemVer方針を文書化する。
- 処理系またはプロファイル更新時のHTMLキャッシュ、検索投影およびグラフ投影の再構築手順を定義する。
- 保存時のサーバ再検証、保護メタデータ、文書内・文書間cross reference、
  逆参照を含むリンク解決およびブラウザプレビューの受け入れ試験を追加する。
- Zed、ブラウザ、サーバおよび将来のTauriで再利用する手動確認項目を整備する。
- 正式公開前にAdocWeave名称、crate、CLI、リポジトリおよびLanguage Server名を再調査する。

== 完了条件

- プロファイル必須構文がすべて縦方向に実装・検証されている。
- サーバとブラウザが同じ入力から同じ診断、projectionおよびHTMLを得る。
- ブラウザ検証を迂回した入力も保存時にサーバで拒否または正規に診断される。
- 互換性を破る変更と再構築が必要な変更を利用者が判定できる。
- アンカーまたは文書識別子の変更時に影響する参照を列挙し、リンク切れを保存前または
  保存時に診断できる。
- 全workspace、WASM、LSP、サーバ統合および受け入れテストが成功する。
