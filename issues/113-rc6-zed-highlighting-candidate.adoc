= RC.6 Zed構文色付け候補の公開と受け入れ
:status: completed
:depends-on: 111

== 目的

RC.5以後に追加したZed Tree-sitter構文色付けとLSP Semantic Tokens責務境界を、stableへ直送せず
`v0.1.0-rc.6`として公開し、実際の配布archiveとmanaged LSPを受け入れる。RC.6をstableの機能baselineとする。

== 作業

* workspace、Zed、browser、Release manifestおよび配布設定を`0.1.0-rc.6`へ同期する。
* 6種類のコア公開契約versionはIssue 111で凍結した値を維持する。
* `cargo make release-gate`、MSRV、依存監査、Linux両architectureのcandidateおよび導入E2Eを完了する。
* mainのcandidate成功後にannotated tag `v0.1.0-rc.6`を作り、protected Environmentを経てprereleaseを公開する。
* 公開9 asset、checksum、metadata、SBOM、attestation、source commitおよびNix flakeを再照合する。
* 公開Zed archiveを隔離profileへ導入し、Semantic Tokensを`off`にしても基本色付けが残ること、`combined`で
  LSPの意味的補足と競合しないことをlight／dark themeで確認する。
* managed LSPの診断、Outline、Code Action、Format、Hover、補完、DefinitionおよびReferencesに回帰がないことを確認する。

== Stableへの凍結

RC.6受け入れ後は、そのtag commitをstable baselineとして記録する。Issue 112ではversion、Release noteおよび
受け入れ記録に必要な変更だけを許可し、新しい構文、API、editor機能または依存更新を追加しない。不具合修正が必要なら
RC番号を進め、このIssueと同じ受け入れを繰り返す。

== 受け入れ記録

2026-07-22に、tag `v0.1.0-rc.6`がsource commit
`6b5686f530ad0cf1c6cbafb658a258bf4f939c0d`を指すことを確認した。tag起点の
https://github.com/KeishiS/AdocWeave/actions/runs/29950271606[CI and Release run 29950271606]は成功し、
prereleaseには予定した9 assetだけが公開された。

公開assetを改めて取得し、次を確認した。

* `sha256sum --check sha256.sum`がarchive 6個、distribution manifestおよびSBOMのすべてで成功した。
* `release-metadata.mjs verify`により、9 asset集合、package version `0.1.0-rc.6`、source commitおよび
  Core Profile 1、Core API 4、HTML 3、Projection 4、Conformance 4、WASM API 4を再照合した。
* 公開API上で9 assetそれぞれにGitHub artifact attestationが1件存在した。
* x86_64 Linuxのclean導入E2EでCLI、LSP、browserおよびZed archiveの導入、version照合と削除が成功した。
  aarch64 Linuxのcandidate・Nix検証は同じtag workflowのmatrixで成功した。
* source commitを固定した公開Nix flakeの`--version --json`がpackage versionと凍結済み契約versionを返した。

残作業は、公開Zed archiveを隔離profileへ導入し、managed LSP、基本色付け、Semantic Tokens、light／dark themeおよび
既存LSP機能を実機で受け入れることだけである。

2026-07-22にNixOS 26.05（x86_64）、Zed 1.10.0で公開Zed archiveを隔離profileへ導入して受け入れた。
managed LSPがRC.6から取得・起動されることを確認した。Semantic Tokensを`off`にしても基本色付けが残り、
`combined`では意味的補足と競合しなかった。One Light／One Darkの双方で構文を判別でき、診断、Outline、
Code Action、Format、Hover、補完、DefinitionおよびFind All Referencesにも回帰はなかった。不一致はない。
この結果により、tag commit `6b5686f530ad0cf1c6cbafb658a258bf4f939c0d`をstable baselineとして凍結する。

== 完了条件

`v0.1.0-rc.6`がprereleaseとして公開され、9 assetと公開flakeが同じsource commit、package versionおよび
凍結済み契約versionを持つ。公開Zed archiveでIssue 110の色付けと既存LSP機能を確認し、stableを阻害する不一致がない。

== 範囲外

新しいAsciiDoc構文、macOS／Windows、registry公開、Zed Extension GalleryおよびRC.5以前のRelease変更は対象外とする。
