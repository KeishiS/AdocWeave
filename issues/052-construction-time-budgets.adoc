= 構築時の処理予算と計測可能な線形性

== 目的

入力から大きな中間表現を構築した後に上限を検査する経路をなくし、CPU・メモリ上限を
処理中に強制する。インライン走査の線形性を計算上の見積りではなく実際の検査回数で
回帰検証できるようにする。

== 作業

* ブロック、semantic node、属性、参照および診断に共通する`ParseBudget`を導入する。
* nodeを確保または結果へ追加する前に予算を消費し、超過地点で`LimitExceeded`を返す。
* 深さ上限と個数上限の責務を同じ解析contextへ集約する。
* InlineScannerの推定inspected_positionsを廃止し、実際の文字・delimiter検査を計測する。
* 密集した不正marker、未閉鎖macro、巨大な属性列および深いリストを回帰入力へ追加する。

== 完了条件

* 上限を超えるnode、block、属性または参照を結果用collectionへ追加しない。
* 上限直前、同値、1超過で決定的に同じ結果になる。
* scanner内部の補助走査を含む実測値が入力長に対する定数倍上限を満たす。
* property test、security test、fuzz targetおよびnative／WASM適合性試験が通る。

== 依存関係

なし。後続の構文木再設計より先に実施する。

M12移行方針により、このIssue単独で旧parserの全consumerとの互換性を維持する必要はない。

== 実装結果

InlineScannerの走査量を入力長から合成した値ではなく、unconstrained marker索引、候補収集、
closer索引およびdelimiter索引で実際に検査した位置の合計へ変更した。ASCIIの既知入力と
不正macroが密集する可変長入力で、計測値そのものと入力長に対する定数倍上限を検証する。

共有ParseBudgetを導入し、文書root、block、semantic node、文書属性およびxrefを結果collectionへ
追加する前に予算を消費するよう変更した。コア解析後のblock、node、属性、参照の事後検査は
削除した。診断も全件生成後のtruncateを廃止し、上限到達後は新しい診断を構築しない。

各構造資源について上限直前、同値および1超過を公開Engine経由で検証し、超過時のresource、
limitおよびactualを固定した。escaped macroは参照nodeを構築せず、参照予算も消費しない。
