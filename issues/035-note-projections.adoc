= メタデータ・参照・検索テキスト抽出

== 目的

HTMLとは独立して、DB投影に必要な情報を同じsemantic ASTから決定的に抽出する。

== 実装範囲

- タイトル、タグ、保護メタデータ、アンカー、ノート参照および通常リンクの抽出型を定義する。
- 検索用可読テキストを生成し、コード、数式および非表示メタデータの扱いを仕様化する。
- 抽出結果に元範囲を保持し、DB更新時の診断と関連付けられるようにする。
- HTMLレンダラーと抽出器が同じASTを走査し、独自の再解析を行わない。
- 抽出結果の順序、重複処理およびプロファイルバージョンを固定する。

== 完了条件

- 一つの解析結果からHTMLと全DB投影を生成できる。
- 同じ入力と設定からバイト単位で安定したシリアライズ結果を得られる。
- リンク切れ検証へ渡す参照集合と本文中の範囲が一致する。
- 検索テキストに保護メタデータや意図しない構文記号が混入しない。
