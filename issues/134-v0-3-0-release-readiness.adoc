= v0.3.0 release readinessとtable表示属性の受け入れ
:status: completed
:depends-on: 133,135

== 目的

`v0.2.0`以後に追加したtable captionと表示属性を、`v0.3.0-rc.N`から`v0.3.0` stableまでの
一つのrelease trainとして受け入れる。公開前に機能追加を停止し、native、WASM、CLI、ZedおよびNixの
成果物が同じversionと契約を参照することを確認する。

== 判断

block title caption、`frame`、`grid`、`stripes`、`width`および`autowidth`は利用者可視の機能追加である。
`0.y.z`期間の方針に従い、次版はpatchではなくminorの`0.3.0`とする。

`CONTRACT_VERSION`は、既存のrequest／response envelopeを変えない限り更新しない。table presentationの
canonical AST表現は同じ単一契約version内の機能追加としてfixtureで固定し、hostは対応versionの成果物を
混在させない。

== 作業

* 最終RCと同じ機能baselineへ、workspace、Zed、browser、release manifestおよび配布設定のversionを`0.3.0`へ同期する。
* `v0.2.0`からの利用者可視差分、HTML allowlist、診断、LSP completionおよびformatter契約を監査する。
* mainのcandidate成功後に`v0.3.0-rc.1`を公開し、9 asset、checksum、attestation、CLI、LSP、browser、ZedおよびNixを受け入れる。
* RC公開後は機能追加を停止し、repository所有者の公開前確認後にstableを公開する。
* blocker修正が公開契約を変更する場合は新しいRCを公開する。stableは最終RCと同じ機能baselineに、stable version metadataだけを同期して公開する。

== 検証

* `cargo make release-gate`
* `node tools/release-contract.mjs --tag=v0.3.0`
* tag workflowの必須job、公開assetの完全性検証およびclean環境の導入E2E
* Linux x86-64／ARM64、browser Worker、NixOSおよびZedの受け入れ記録

== 受け入れ結果

`v0.3.0-rc.1`は`f3503083f1caa051266cce5e7376535599e0d9df`、stable `v0.3.0`は
`4e08c0334ee65d68e3d0cfa170a65aab5a9a3b14`をそれぞれtagした。RC Releaseはprerelease、stable Releaseは
非draft・非prereleaseであり、いずれも計画どおり9 assetを含む。

stableのhttps://github.com/KeishiS/AdocWeave/actions/runs/30015154848[Release run 30015154848]は、
quality、Linux x86-64／ARM64 native build・smoke、browser／Zed artifact verification、candidate metadata、
Linux 2 architectureのNix packageを含むinstallation E2Eおよびpublishをすべて成功した。
公開asset集合はCLI／LSP各2 architecture、browser、Zed、manifest、SPDX SBOM、checksumの9 fileであり、
0 byte assetはなかった。`cargo make release-gate`と`node tools/release-contract.mjs --tag=v0.3.0`も成功した。

`v0.2.0`からの機能差分はtable caption、`frame`、`grid`、`stripes`、`width`および`autowidth`である。
公開契約versionは2のままとし、移行手順はxref:../docs/upgrade-0-3.adoc[v0.3.0への移行]へ記録した。
