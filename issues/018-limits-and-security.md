# 上限とセキュリティ制約

## 目的

初期リリースの入力に対する資源上限と安全性を実装する。

## 実装範囲

- 入力10 MiB、出力50 MiB、1行1 MiBの初期上限を設定する。
- リスト深さ8、インライン深さ32、属性数1,000および診断数の上限を設定できるようにする。
- 生HTMLを通過させない回帰テストを追加する。
- コアがファイル、ネットワーク、環境変数、時刻および外部コマンドへアクセスしないことを設計上明確にする。
- 厳格モードと寛容モードの未対応構文の扱いを定義する。
- 入力本文を通常ログへ出力せず、panicをライブラリ境界外へ契約として伝播させない。

## 完了条件

- 各上限の直前、同値、超過をテストできる。
- 上限超過時にpanicや部分的な危険出力が発生しない。
- 極端な入力に対して概ね入力サイズに比例して処理できる。
- 同じ入力と設定でAST、診断、HTMLが決定的である。
- リスト深さと属性数は公開設定に予約し、対象構文を導入するIssueで実際のノード生成時に強制する。

## 検証

```console
cargo test limits
cargo test security
```
