= LaTeX STEMと数式拡張境界

== 目的

LaTeXによるインライン・ブロックSTEMを解析し、将来のTypst数式と共存できる拡張境界を定義する。

== 実装範囲

- `stem` 文書属性、インラインSTEMおよびブロックSTEMの文法と終端規則を定義する。
- 数式本文を別言語として扱い、AsciiDocインライン置換を適用しない。
- 未閉鎖数式、空数式、サイズ上限および入力途中の回復を実装する。
- ASTで数式言語と元範囲を明示し、LaTeXと将来のTypstを列挙可能にする。
- コアは数式を実行・評価せず、レンダラーまたはホストへ安全な構造として渡す。
- 数式レンダラー出力を独立した信頼境界として扱う契約を文書化する。

== 完了条件

- 通常テキスト、属性および数式の置換境界がfixtureで固定されている。
- 編集用寛容モードで未閉鎖数式を含むASTと診断を返せる。
- HTMLレンダラーが入力を任意HTMLとして通過させない。
- Typst追加時に既存LaTeXノードの意味を変えないAPI境界がある。

== 検証

[source,console]
----
cargo test --workspace stem
cargo test --workspace stem_recovery
----

== 実装記録

完了。

- `stem`文書属性、`stem:[...]`/`latexmath:[...]`および`[stem]`/`++++`ブロックを
  解析する。
- `InlineFormula`と`MathBlock`がLaTeX言語、元範囲および不透明な本文を保持する。
- 空、未閉鎖、1 MiB超過を`invalid-stem`で診断し、未閉鎖インラインにもASTを返す。
- HTMLは数式本文を`code`/`pre`内でエスケープし、評価やraw HTML通過を行わない。
- `MathLanguage`を閉じた意味variantとして定義し、将来のTypst追加がLaTeXの意味を
  変更しない境界にした。
- 置換fixtureと信頼境界を`docs/stem.adoc`および規範的置換表へ記録した。
