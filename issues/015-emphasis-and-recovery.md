# 斜体とインラインエラー回復

## 目的

斜体構文を追加し、基本インライン構文全体のエラー回復を確立する。

## 実装範囲

- 斜体記号の開始・終了条件を仕様化する。
- 子要素を持つ `Inline::Styled { style: Emphasis }` を追加し、HTMLの `em` へ変換する。
- インラインASTを、置換を行わない `Literal` と子要素を再帰的に解釈する `Styled` に分離する。
- 区切り記号の検出、境界判定、深さ管理および回復を出力バックエンドから独立させる。
- 識別子内のアンダースコアなど曖昧な入力を通常文字として扱う。
- 等幅、強調、斜体の組合せと深さ上限を検査する。
- 未閉鎖構文から次の安全な境界へ回復する。

## 完了条件

- 基本インライン構文の組合せが決定的に解析される。
- 未閉鎖構文が後続段落や見出しを壊さない。
- 深さ上限超過を `nesting-limit-exceeded` として報告する。
- 対応中の領域をFormatterが変更しない。
- HTML以外の出力バックエンドがタグ固有情報なしに同じインラインASTを走査できる。

## 検証

```console
cargo test emphasis
cargo test inline_recovery
```
