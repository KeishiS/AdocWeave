# 入力行と改行の認識

## 目的

元テキストの範囲を失わずに、行と空行を認識する。

## 実装範囲

- LFとCRLFを改行として認識する。
- 各行について本文、改行、全体の範囲を保持する。
- 空入力、空行、末尾改行の有無を区別する。
- 混在改行を受理し、後続のLintまたはFormatterが検出できる情報を残す。
- 空白、コメント、区切り記号および未対応領域を元バイト列のまま保持できるlossless token列の骨格を実装する。

## 完了条件

- 行の範囲から元の入力を復元できる。
- token列を連結すると元の入力とバイト単位で一致する。
- CRLFの途中を有効な位置として扱わない。
- 1 MiBの単一行を境界値fixtureとして扱える。

## 検証

```console
cargo test source_lines
```
