= リアルタイムHTML変換用browser library
:status: completed
:depends-on: 082

== 目的

現在別々に存在する`wasm-bindgen`成果物と`web-worker` sourceを、frontendがAsciiDoc入力をリアルタイムで
HTMLへ変換するためのversion付きbrowser libraryへする。library自身はDOMを変更せず、最新generationの
HTMLと診断等を利用側へ返す。

== 作業

* `cargo make build-wasm`をrelease packaging用の決定的な出力directoryとfile名へ整理する。
* `.wasm`、ES module glue、TypeScript declaration、`client.mjs`、`controller.mjs`、`worker.mjs`、契約定数、
  `package.json`、license、READMEおよび最小HTML例を固定layoutでarchiveする。
* frontendが直接Worker protocolを組み立てなくてよいES module入口を定義する。初期化、source更新、
  monotonic generation、最新結果だけの採用、cancel、disposeおよびerrorを型付きAPIとして公開する。
* 変換結果は少なくとも`html`、診断、source version、generationおよび契約versionを所有する。
  libraryは`innerHTML`等でDOMへ挿入せず、描画方法と追加sanitizationは利用側が決定する。
* 高頻度入力ではdebounceと協調cancelを使い、古いHTMLを最新入力として通知しない。Worker再生成による
  `SharedArrayBuffer`なしの縮退経路も同じ公開APIの内側へ隠す。
* debug path、timestamp、absolute pathおよび不要なNode向け成果物を公開bundleへ混入させない。
* `release-manifest.json`、WASM response、Worker runtime定数およびbundle versionを一つの検査で照合する。
* 実際のarchiveを展開し、静的HTTP serverとheadless browserでWorker初期化、解析、cancel、CSP、
  cross-origin isolation利用可否の縮退経路をsmoke testする。
* Node用cross-runtime fixtureは意味同値性、browser testは配布layoutとloaderの検証として役割を分ける。
* raw `.wasm`やWorker messageを公開入口とせず、bundle内READMEからversion付きbrowser APIを利用する。
* bundlerなしのES module読込と、代表的なfrontend bundlerの双方でasset URL解決を試験する。

== 完了条件

browser library archiveだけを新しいdirectoryへ展開し、textareaへの連続入力から最新HTMLだけをpreviewへ
描画する最小例が動作する。native／WASM conformance、実browserでの高速連続更新、cancel、型定義の
compile check、CSPおよびbundle size上限が通る。

== 実装状況

`adocweave-browser-{version}.tar.xz`は決定的なlayoutで、WASM、ES module glue、TypeScript宣言、
公開`worker/index.mjs`、内部Worker実装、契約定数、package metadata、license、third-party noticeおよび
textarea最小例を含む。WASM buildはworkspaceとCargo homeをremapし、archive検査は機械固有の絶対pathを拒否する。

`AdocWeaveClient`はWorkerの生成と初期化、source update、単調generation、debounce、最新結果だけの通知、
cancel、disposeおよびerror通知を所有する。結果はHTML、解析・render診断、source version、generation、
6契約versionおよび完全なWASM応答を所有し、DOMを変更しない。cross-origin isolated環境では
`SharedArrayBuffer`による協調cancelを使い、それ以外ではWorker終了・再生成へ同じAPI内で縮退する。

`cargo make test-browser-release-package`はlayout、決定性および機械固有path不在を検査する。
`test-browser-smoke`はarchiveを一時directoryへ展開し、静的HTTP serverとChromiumでCSP下の非isolated／isolated
両経路、高頻度更新、cancel、最新HTML、WASM応答とrelease manifestの契約一致、2 MiBのarchive上限および
1 MiBのWASM上限を検査する。`test-browser-bundler`は公開入口をesbuildでbundleした同じ利用例を再検査する。
TypeScript compileとnative／WASM共通fixtureの意味同値性もrelease gateへ含めた。
