= リストとブロック継続
:status: completed
:depends-on:

== 目的

順序なし・順序付きリスト、入れ子およびブロック継続をlosslessに解析する。

== 実装範囲

- リストマーカー、マーカー後の空白、インデント、入れ子および終了条件を規範化する。
- 順序なし・順序付きリストと項目をCST・ASTへ追加する。
- ブロック属性行、継続記号および後続ブロックとの関連付けを定義する。
- リスト深さ上限8と `inconsistent-list` 診断を実装する。
- `ul`、`ol` および `li` のHTML契約、文書シンボル、診断および安全な空白Fixを追加する。

== 完了条件

- 混在マーカー、不正な入れ子、不完全項目および深さ超過から回復できる。
- リスト内部のコード・リテラルブロックを継続規則に従って保持できる。
- Formatterが意味を変えず、未対応の継続領域を保存する。
- 対応範囲のAsciidoctor差分fixtureがある。

== 検証

[source,console]
----
cargo test --workspace lists
cargo test --workspace list_continuation
----

== 実装記録

完了。

- 順序付き・順序なしリストを再帰的な`ListBlock`/`ListItem` ASTへ変換した。
- リテラルブロックとソースブロックへの`+`継続を項目へ関連付けた。
- `inconsistent-list`診断と、タブ区切りに対する安全な空白修正を追加した。
- HTMLの`ul`/`ol`/`li`、文書シンボル、Formatterによる原表記保持を追加した。
- 対応範囲と未対応範囲を`docs/lists.adoc`へ記録した。
