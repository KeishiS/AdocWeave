= Zed拡張による公開LSP binaryの解決
:status: completed
:depends-on: 086

== 目的

Zed拡張が開発時の`PATH`優先動作を維持しつつ、通常導入時にはGitHub Releaseから対応platformの
`adocweave-lsp`を安全に取得、検証、cacheおよび更新できるようにする。

== 作業

* Zed extension APIのLinux platform／architectureをRust targetとassetへ写像し、macOS、Windows、32-bitと
  未知platformを、初回公開では明示的なunsupported errorにする。
* user設定またはworktreeの`PATH`にある明示binaryを最優先し、次にextensionが管理するversion一致binaryを
  使う。暗黙にsystem binaryとdownload binaryを混在させない。
* GitHub Releaseのdistribution manifestからexact extension versionのLSP assetとSHA-256を選択し、
  temporary fileへdownload、hash検証、展開、permission設定、atomic renameの順でinstallする。
* Zed extensionの`work` directoryだけへ保存し、project worktree、`PATH`および任意directoryへ書き込まない。
* download中断、hash不一致、offline、rate limit、未対応platformおよびread-only filesystemを安定した
  user-facing errorとlogへ変換する。
* extension versionとLSP versionを同じrelease trainで固定し、更新失敗時は検証済み旧binaryを保持する。
* dev extensionでは従来どおりlocal `target/debug/adocweave-lsp`を検証できる手順を残す。

== 完了条件

対応するLinux platformでcleanなZed user dataからLSPが自動取得・起動し、hash不一致を実行前に拒否する。PATH override、
offline cache、更新rollbackおよび全失敗modeをadapter testとZed手動試験で確認する。

== 実装状況

Zed設定、worktreeの`PATH`、version一致managed binaryの順に解決するadapterを実装した。Linux x86-64／ARM64の
target写像、exact tagとdistribution manifestによるasset選択、byte数／SHA-256の展開前検証、上限付きの純Rust
XZ/TAR抽出、検証marker付きcacheおよびatomic更新rollbackを備える。単体testはmanifest不一致、hash不一致、抽出、
cache改変およびcommit失敗時の旧cache復元を検査する。規範契約は
xref:../docs/zed-lsp-resolution.adoc[zed-lsp-resolution.adoc]に記載した。

`wasm32-wasip2` buildとadapter testに加え、2026-07-22に公開`0.1.0-rc.1`からLinux x86-64用LSPを
Zedの管理directoryへ取得し、`verified.json`とbinaryが生成されることを確認した。Zed上でLSPが起動し、
Hoverと診断が表示された。offline時に利用する検証済みcache、改変拒否、PATH overrideの優先順位および
更新失敗時のrollbackは同じ解決・install境界の自動testで固定した。
