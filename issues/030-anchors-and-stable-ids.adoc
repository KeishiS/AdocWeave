= 明示アンカーと安定ID

== 目的

文書内および文書間cross referenceの対象にできる明示アンカーと、互換性を管理できる
自動見出しIDを実装する。

== 実装範囲

- 明示アンカーの構文、文字集合、エスケープおよび対象ブロックとの関連付けを規範化する。
- Unicode正規化を行う箇所と、元表記を保持する箇所を定義する。
- 自動見出しIDの生成、空ID、衝突および重複時の挙動を定義する。
- 明示IDを自動IDより優先し、安定したアンカー索引をASTから生成する。
- 文書タイトル、見出しおよび明示アンカーから、参照対象の種別、表示名、IDおよび
  ソース範囲を持つ共通 `ReferenceTarget` を生成する。
- 重複または不正アンカーを位置付きで診断する。

== 完了条件

- 同じ入力とプロファイルから同じアンカー索引を生成する。
- 日本語、結合文字、絵文字および大小文字を含む境界fixtureがある。
- HTMLのIDと参照解決用IDが仕様どおり一致する。
- 後続の文書内参照と文書間参照が同じアンカー索引を利用できる。
- 見出しID規則の変更を出力契約の変更として検出できる。

== 実装記録

完了。

- `[[id]]`、`[[id,label]]` および `[#id]` をlosslessなブロックアンカーとして
  解析し、直後のブロックへ関連付ける。
- 明示IDはUnicode正規化と大小文字変換を行わず、元表記を意味上のIDとして使用する。
- 明示アンカーを自動見出しIDより優先し、明示IDを予約した上で自動IDの衝突を
  決定的な連番で回避する。
- 文書タイトル、セクションおよび明示アンカーを共通の `ReferenceTarget` 索引として
  生成する。種別、表示名、ID、ID範囲および対象範囲を保持する。
- HTMLは同じ見出しID生成結果とアンカー索引を使用し、見出し以外の対象ブロックにも
  明示IDを出力する。
- 不正、対象なし、同一対象への複数指定を `invalid-anchor`、重複IDを
  `duplicate-anchor` として診断する。
- 日本語、結合文字、絵文字および大小文字の境界fixtureを追加した。
- 規則を `docs/anchors.adoc` に記録した。
