= 文書全体のidentifier catalog統一
:status: completed
:depends-on: 090

== 目的

明示anchor、自動見出しIDおよびinline anchorの決定を文書全体で一度だけ行い、HTML、xref、LSP、projectionと
診断が同じidentifier catalogを参照する。

== 背景

現在の`reference_targets`はtop-level見出し、nested blockの明示anchor、inline anchorを複数の走査で追加する。
HTMLはcatalogに自動IDがないnested headingに対して`heading_id_base`を直接呼ぶため、文書全体の衝突回避と
xref対象集合が一致しない可能性がある。

== 作業

* 全階層の見出しとanchorをsource順に収集する`DocumentIdentifiers`をlowering時に一度構築する。
* 明示IDの予約、自動ID生成、衝突suffix、重複診断およびID rangeを一つのalgorithmへ統合する。
* `DocumentStructure`はcatalogのtop-level構造projectionとし、IDを独自生成しない。
* HTML rendererの`heading_id_base` fallbackを削除し、全見出しのIDをcatalogから受け取る。
* xref解決、LSP、WASM／JSON projectionおよびconformanceを同じcatalogへ移行する。
* top-level、compound、list continuation、AsciiDoc cell間のID衝突とnested headingへのxrefを試験する。

== 完了条件

描画可能な全見出しに文書内で一意な決定済みIDがあり、rendererやconsumerがIDを再生成しない。HTMLの`id`、
参照先一覧、xref解決およびLSP navigationが同じIDとrangeを返す。catalog外のfallback生成経路が存在しない。

== 完了結果

`AstDocument`へ不変`DocumentIdentifiers`を追加し、lowering後に全階層のblock／inline anchorと見出しをsource順で
一度だけ登録するようにした。明示IDとinline anchorを先に予約し、自動見出しIDのsuffixを文書全体で決定する。
`DocumentStructure`はtop-level見出しの階層だけを投影し、IDを生成しない。

`Analysis`が所有していた重複`reference_targets`を削除し、HTML、xref、LSP、projectionはAST内catalogを参照する。
HTML rendererの`heading_id_base` fallbackも削除した。top-levelとcompound内で同名の見出しを持つfixtureにより、
`_same`／`_same_2`、nested xrefおよびHTML `id`の一致を固定した。
