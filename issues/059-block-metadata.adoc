= 共通block metadata
:status: completed
:depends-on: 058

== 目的

blockごとの個別処理を、公式仕様のblock attribute、anchor、title、ID、roleおよびoptionを表す
共通metadata modelへ置き換える。

== 作業

* 隣接するmetadata行と対象blockの結合規則をsyntax treeへ追加する。
* positional／named attribute、shorthand ID／role／optionをlosslessに認識する。
* semantic blockへ出力非依存なmetadataをloweringする。
* unknown styleと属性を保持し、HTML属性として無条件に通過させない。
* Formatter、diagnostic、LSP hover／completionおよびcanonical fixtureを更新する。

== 完了条件

複数metadata行の順序と範囲を復元でき、すべての対応blockが同じquery APIを利用する。

== 参照

https://docs.asciidoctor.org/asciidoc/latest/blocks/[Blocks]および
https://docs.asciidoctor.org/asciidoc/latest/attributes/element-attributes/[Element Attributes]。

== 実施結果

`BlockMetadata`を全`AstBlock` variantへ統合し、block title、明示anchor、shorthand
ID／role／option、positional／named属性を値とsource range付きでloweringするようにした。
syntax treeは隣接する複数metadata行をannotationとして元順序のまま保持し、孤立した列は
`Unsupported`へlosslessに回復する。

共通walker、位置query、LSP hover／completionおよびcanonical ASTから同じmetadataを取得できる。
Formatterはmetadata行をbyte単位で保護し、HTML rendererは未知role、option、styleおよび属性を
通過させない。公開型変更により`CORE_API_VERSION`を11、canonical schemaを2、
`CONFORMANCE_CONTRACT_VERSION`を6へ更新した。
