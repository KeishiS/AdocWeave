= CLIとLSPのnative release artifact
:status: planned
:depends-on: 082

== 目的

各対応targetについて、repository checkoutやRust toolchainを要求しないCLIとLanguage Serverのarchiveを
再現可能な手順でbuildし、展開後に直接実行できる状態にする。

== 作業

* release profileのpanic、LTO、codegen units、stripおよびdebug symbol方針を明示する。
* CLIとLSPを別assetとしてbuildし、archive root、executable permission、`.exe`、license、READMEおよび
  completion等の同梱規則を固定する。
* `MIT OR Apache-2.0`に対応するlicense本文とthird-party noticeをrepositoryの正本として追加し、全archiveへ
  同じfileを同梱する。Cargo metadata上のlicense表記だけで配布を済ませない。
* Linuxはmusl static linkを検証し、`ldd`または同等の検査で想定外のdynamic dependencyを拒否する。
* Linux 2 targetをbuildし、各architectureの実行環境で検証する。cross compileだけで「対応」と判定しない。
* archiveを新しいtemporary directoryへ展開し、CLIの`--version`、`check`、`convert`、`format --check`と、
  LSPのinitialize、fixture診断、shutdown／exitをbinary経由でsmoke testする。
* executableが報告するpackage version、Core contractおよびLSP server versionをrelease manifestと照合する。
* macOS／WindowsのbuildおよびOS code signingは初回公開の対象外とし、Linux公開後の独立milestoneで
  credential ownerと運用を決定する。将来も鍵をrepositoryへ置かない。

== 完了条件

Linux 2 targetのCLI／LSP archiveをlocalまたはPR artifactとして生成でき、対応architecture上で展開後smoke testが通る。
host toolchainや作業tree内binaryを誤って試験していないことをpath検査で証明する。
