= cross reference解決とnote参照Resolver

== 目的

標準cross referenceと`note:<uuid>#<anchor>[label]`を同じ参照モデルで扱い、
DB・ファイルシステム非依存のホスト注入型Resolver境界を提供する。

== 実装範囲

- UUID、任意アンカー、表示文字列、エスケープおよび不完全入力の文法を定義する。
- Issue 031の `Reference` を拡張し、相対文書表記、ノートID、アンカーおよび表示文字列を
  共通の問い合わせへ変換する。
- 構文検証、対象文書の識別、アンカーの存在確認および最終href生成を別段階にする。
- 文書候補検索、文書識別、UUID解決、アンカー解決および逆参照検索の非同期
  Resolverインターフェースをホスト層に定義する。
- コア解析はResolverを呼ばず、補完、ホバーおよび定義移動の問い合わせ情報だけを返す。
- Resolverへ渡す文書コンテキストは不透明な `SourceId` とし、コア型へDB接続、
  ワークスペースルートまたはOSパスを持ち込まない。
- 不正UUID、対象文書不在、アンカー不在、曖昧な対象、ルート外参照およびResolver障害の
  安定した診断コードを定義する。
- 解決結果のキャッシュキーと無効化単位を、参照元、参照先、プロファイルおよび
  文書versionから構成できるようにする。

== 完了条件

- Resolverなしでも構文解析、HTML生成および参照抽出が決定的に動作する。
- Resolver結果を信頼せず、HTML出力前にURLポリシーを適用する。
- 同一文書、別文書および`note:`参照が同じ解決結果型と診断経路を利用する。
- Resolverが逆参照を提供する場合、参照元範囲を失わずにreferences一覧を返せる。
- 入力中のマクロから過度に点滅する診断を出さない寛容モードfixtureがある。
- CLI、LSP、サーバおよびWASMが同じ参照型を利用できる。

== 実装記録

完了。

- `note:`をxref参照先スキームとして維持する。実装当時は任意登録する
  `ReferenceSchemeExtension`がcanonical lowercase UUIDを検証した。
- 別文書、noteおよび任意スキームを共通の`ReferenceQuery`/`ReferenceKey`へ正規化した。

== 後続の破壊的変更

Issue 044で固定的な`ReferenceSchemeExtension`とUUID制約をコアから除去した。
未知スキームは引き続き汎用参照として保持し、locator検証は解析後の外部ホスト拡張が
行う。
- 文書候補、文書、任意スキーム、アンカーおよび逆参照の非同期
  `ReferenceResolver`境界を追加した。
- Resolver失敗の分類、安定診断コード、逆参照範囲およびキャッシュキーを公開型にした。
- Resolverなしの解析・抽出・HTML回復と、解決hrefへのURLポリシー再検査を維持した。
- 規範契約を`docs/reference-resolution.adoc`へ記録した。
