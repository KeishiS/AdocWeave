# 規範的文法と置換モデル

## 目的

実装を一意にするため、対応済み構文の字句、境界、優先順位、回復および置換を規範的に定義する。

## 実装範囲

- 文書、段落、見出し、等幅、強調、斜体、リテラルブロックおよびソースブロックのEBNFまたは同等の規則を文書化する。
- constrained・unconstrained形式、エスケープ、入れ子優先順位および開始・終了条件を定義する。
- 通常文、属性、コード、リテラル、リンクおよび数式のコンテキスト別置換表の骨格を定義する。
- semantic ASTの `Text`、`Literal`、`Styled` と置換表の対応を定義し、出力形式固有の表現を規範文法から分離する。
- ブロック属性行と対象ブロックの関連付け、空行およびインデントの共通規則を定義する。
- 不完全入力、曖昧な入力および `Unsupported` への回復規則をfixtureで固定する。

## 完了条件

- 対応構文の各ASTノードを規範的規則へ追跡できる。
- 曖昧な入力について期待するCST、ASTおよび診断が明示されている。
- 既存fixtureが規則と矛盾しない。
- 以後の構文Issueが参照する置換表と回復規則の形式が定まっている。
- 新しい出力バックエンドが解析や置換を再実装せずsemantic ASTだけを入力にできる。

## 検証

```console
cargo test --workspace grammar
cargo test --workspace substitutions
```
