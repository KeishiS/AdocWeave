= 文書属性と保護メタデータ

== 目的

ノートアプリ互換プロファイルに必要な文書属性を解析し、安全に参照・検証できるようにする。

== 実装範囲

- 文書タイトル直後の属性定義、属性解除および単純な属性参照を規範化する。
- `note-id`、`creator-id`、`created-at`、`updated-at`、`tags` および `stem` を型付きメタデータとして扱う。
- UUID、日時およびタグの字句・正規化規則を定義する。
- 保護属性と利用者編集可能属性をプロファイル設定で区別する。
- 再帰展開、条件分岐および外部環境参照を `Unsupported` とする。
- `undefined-attribute`、`unused-attribute`、重複および保護属性変更の診断を追加する。

== 完了条件

- CSTが属性行の元表記を保持し、ASTが型付き値と参照を保持する。
- 保存用厳格モードでは不正または変更禁止の保護属性をエラーにできる。
- 編集用寛容モードでは不完全な属性から回復できる。
- 属性補完、ホバーおよび安全な表記統一に必要な情報をコアAPIから取得できる。

== 検証

[source,console]
----
cargo test --workspace document_attributes
cargo test --workspace protected_metadata
----

== 実装記録

完了。

- 文書タイトル直後の定義 `:name: value`、解除 `:name!:`・`:!name:` と、
  通常インライン文脈の `{name}` を実装した。
- CSTは属性行の原表記を保持し、ASTは名前・値の独立範囲、操作、原値および
  型付き値を保持する。
- `note-id`、`creator-id`、`created-at`、`updated-at`、`tags` および `stem`
  の字句検証と正規化を実装した。
- `ParseOptions.protected_attributes` により、ホストが保護値を注入できる。
  変更または解除は寛容モードで警告、厳格モードでエラーになる。
- `invalid-attribute`、`duplicate-attribute`、`undefined-attribute`、
  `unused-attribute` および `protected-attribute` を追加した。
- 属性参照は一回だけ置換し、再帰展開、条件分岐および外部環境参照を実装しない。
- 属性数に `ProcessingLimits.max_attributes` を適用した。
- 構文、型、保護属性および対象外機能を `docs/document-attributes.adoc` に記録した。
