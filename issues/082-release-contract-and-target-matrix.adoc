= GitHub Releasesの配布契約とtarget matrix
:status: planned
:depends-on: 094

== 目的

GitHub ReleasesからCLI、Language Serverおよびbrowser変換libraryを安全かつ機械的に取得できるよう、
tag、version、asset名、対応platform、archive内容およびmanifest schemaを先に固定する。

このリポジトリに独立したGUI applicationは存在しない。ここでいうfrontend向けlibraryは、利用側の
AsciiDoc入力をWeb Workerで継続解析し、最新generationの安全なHTML、診断およびprojectionを返す
`adocweave-wasm`と`web-worker`のbrowser packageを指す。DOM描画、UI component、editor stateおよび
ノートアプリ本体は利用側リポジトリの責務とする。Zed拡張はfrontend libraryではなくLSP consumerである。

== 配布単位

一つのGit tag `v{packageVersion}`から、同じversionを持つ次のassetを生成する。

* `adocweave-{version}-{target}`：CLI executableとlicense／README
* `adocweave-lsp-{version}-{target}`：LSP executableとlicense／README
* `adocweave-browser-{version}`：browser向けWASM、JavaScript API、TypeScript declaration、Worker module、
  licenseおよびリアルタイムHTML変換の最小利用例
* `adocweave-dist-manifest.json`：asset名、target、kind、byte数、SHA-256、executable名、package version、
  Core／WASM等の契約version
* `SHA256SUMS`、SBOMおよびprovenance verification手順

GitHubが自動生成するsource archiveは上記binary contractへ含めない。assetはrelease後に上書きせず、
修正は新しいversionで公開する。

== 初期target

[cols="3,2,3",options="header"]
|===
|Rust target |archive |目的

|`x86_64-unknown-linux-musl`
|`.tar.xz`
|glibc versionに依存しないLinux x86-64

|`aarch64-unknown-linux-musl`
|`.tar.xz`
|Linux ARM64

|===

初回公開は上記2種類のLinux targetだけを対応対象とする。32-bit targetとGNU Linux dynamic buildは扱わない。
targetの追加・削除は配布契約変更としてmanifestと文書を同時更新する。

== 将来target

次のtargetは初回公開の完了条件へ含めない。Linux releaseの運用、利用要求、native runner、署名credentialおよび
OS固有試験を確認した後に、独立したmilestoneで追加する。

[cols="3,2,3",options="header"]
|===
|Rust target |archive |目的

|`x86_64-apple-darwin`
|`.tar.xz`
|Intel macOS

|`aarch64-apple-darwin`
|`.tar.xz`
|Apple Silicon

|`x86_64-pc-windows-msvc`
|`.zip`
|64-bit Windows
|===

Windows ARM64はWindows x86-64の公開後に利用要求を確認して判断する。

== 初回公開時の契約version baseline

開発中に蓄積した番号を、初回の公開互換性履歴として外部へ持ち出さない。最初のrelease candidateを作る変更で、
`CORE_PROFILE_VERSION`、`CORE_API_VERSION`、`HTML_CONTRACT_VERSION`、
`PROJECTION_CONTRACT_VERSION`、`CONFORMANCE_CONTRACT_VERSION`および`WASM_API_VERSION`を
すべて`1`へ一度だけリセットする。これはworkspace packageのSemVerとは別の操作である。

同じ変更でRust定数、`release-manifest.json`、fixture、Worker、テストおよび現在値を記載する文書を更新し、
release gateで不一致を拒否する。リセット後は各契約を再び独立に単調増加させ、公開済み番号の再利用や巻き戻しを
行わない。完了済みIssueに記録された開発中の履歴は書き換えず、公開前の履歴であることをこのbaseline定義により
区別する。

== 作業

* workspace crate、`editors/zed/extension.toml`、`release-manifest.json`およびbrowser libraryのversionを
  一つのrelease trainとして検査する。
* workspace packageへrepository、homepage、license fileおよびbinary配布に必要なmetadataを追加し、
  `cargo metadata`と`dist`が同じrepository／versionを解決することを検査する。
* stable tag `vX.Y.Z`とprerelease tag `vX.Y.Z-rc.N`だけを受理し、tagと全versionの不一致をbuild前に
  拒否する。
* native archive生成には`dist`（旧cargo-dist）をexact versionでpinして用いる。生成workflowと設定は
  repositoryへcommitし、更新時に差分をreviewする。
* browser libraryは`dist`のextra artifactとして同じreleaseへ統合し、別のversionや手動upload経路を作らない。
* 初回release candidateの作成前に、上記6契約を一括で`1`へリセットする原子的な変更を適用する。
* `adocweave-dist-manifest.json`のschema、sort順、path separator、hash対象およびJSON canonicalizationを
  fixtureで固定する。
* crates.io、npm、Homebrew、WinGetおよびZed Extension Registryへの公開は非目標とし、GitHub assetが
  それらの将来backendからも参照できる命名にする。

== 完了条件

release tagから生成される全asset名と内容がmachine-readable fixtureで決定的に再現され、未対応platform、
version関係、初回契約version baseline、後方互換性および非目標が文書化される。初回公開対象がLinux 2 target
だけであることをmanifest fixtureで検査し、`dist plan`相当のdry runをPRで実行できる。
