= WASM APIとWeb Worker統合

== 目的

ブラウザでもネイティブと同じRustコアを利用し、入力中の解析をUIスレッドから分離する。

== 実装範囲

- parse、render、診断、シンボルおよびprojectionを公開するバージョン付きWASM APIを定義する。
- JS境界の入力、出力、エラーおよびキャンセル表現を安定したスキーマにする。
- Web Worker内でWASMをロードし、短いdebounce、文書バージョンおよびキャンセルを実装する。
- 古い解析結果をUIへ返さないメッセージプロトコルを定義する。
- 初期ロード時間、バンドルサイズ、メモリ上限および大きなノートでの縮退動作を計測する。
- CSPと数式レンダラー等の別信頼境界を統合手順に明記する。

== 完了条件

- UIスレッド上で解析を実行しない。
- 高頻度の変更後に最新バージョンの結果だけを通知する。
- WASM側でも同じ構文プロファイル、上限、診断コードおよびHTML契約を使う。
- 将来TauriからネイティブRustまたは同じWASM APIを再利用できる。

== 検証

[source,console]
----
cargo test --workspace wasm_api
npm test -- web-worker
----
