= WASM APIとWeb Worker統合

== 目的

ブラウザでもネイティブと同じRustコアを利用し、入力中の解析をUIスレッドから分離する。

== 実装範囲

- parse、render、診断、シンボルおよびprojectionを公開するバージョン付きWASM APIを定義する。
- JS境界の入力、出力、エラーおよびキャンセル表現を安定したスキーマにする。
- Web Worker内でWASMをロードし、短いdebounce、文書バージョンおよびキャンセルを実装する。
- 古い解析結果をUIへ返さないメッセージプロトコルを定義する。
- 初期ロード時間、バンドルサイズ、メモリ上限および大きなノートでの縮退動作を計測する。
- CSPと数式レンダラー等の別信頼境界を統合手順に明記する。

== 完了条件

- UIスレッド上で解析を実行しない。
- 高頻度の変更後に最新バージョンの結果だけを通知する。
- WASM側でも同じ構文プロファイル、上限、診断コードおよびHTML契約を使う。
- 将来TauriからネイティブRustまたは同じWASM APIを再利用できる。

== 検証

[source,console]
----
cargo test --workspace wasm_api
npm test -- web-worker
----

== 実装結果

`adocweave-wasm` crateにversion 1の所有型APIを追加し、一つの`Analysis`からparse概要、
HTML、診断、シンボルおよびprojectionを返す。未知フィールド、API version、コア上限および
キャンセルを安定したエラーとして扱い、native契約テストと`wasm32-unknown-unknown`の
型検査を追加した。

`web-worker`にはversion 1のclient、Workerおよび純粋なcontrollerを追加した。debounce、
文書version、generation、古い結果の破棄を実装し、SharedArrayBufferのgenerationを
WASM callbackから読むことで同期解析中の協調キャンセルを可能にした。

ビルド、bundle size測定、メモリ・大文書の縮退、CSP、cross-origin isolationおよび
数式レンダラーの信頼境界はxref:../docs/wasm-worker.adoc[WASM APIとWeb Worker統合]に
記載する。
