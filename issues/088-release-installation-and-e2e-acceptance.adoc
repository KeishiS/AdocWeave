= Release導入文書とend-to-end受け入れ
:status: planned
:depends-on: 086,087

== 目的

GitHub Releaseから初めて導入する利用者とfrontend integratorが、source checkoutなしでassetを選択、検証、
install、更新および削除できることを最終的に確認する。

== 作業

* OS／architectureから正しいCLI・LSP assetを選ぶ表と、checksum／attestation検証手順を追加する。
* user-local install先、system-wide install、PATH、shell completion、更新、version固定およびuninstallを
  platform別に記載する。`curl | sh`を唯一の手順にしない。
* browser libraryの静的hosting、ES module／WASM MIME type、CSP、cross-origin isolation、cache busting、
  integrity確認およびリアルタイムHTML previewの利用例をfrontend向け文書へ追加する。
* Zed Gallery導入、binary cache、PATH override、offline動作および完全削除を開発版手順と分離する。
* GitHub prereleaseをclean Linux x86-64、Linux ARM64、browser frontendおよびLinux版Zed profileへ導入する
  E2E checklistを実行する。macOS／Windowsは将来targetの受け入れIssueへ分離する。
* release noteに対応target、契約version、breaking change、既知の制約、asset verificationおよびupgrade手順を
  必須化する。
* package manager、container image、crates.io、npmおよびZed Registry公開は個別Issueへ分離し、GitHub
  binary releaseの合否へ混ぜない。

== 完了条件

source checkoutとRust toolchainのないclean環境でCLI、LSP、browser libraryおよびZed連携を確認し、install前後の
filesystem差分からuninstall後に管理対象fileが残らないことを確認する。最終手順を
`docs/release-checklist.adoc`と利用者向け導入文書へ反映する。
