= Source map付き解析projection
:status: completed
:depends-on: 067

== 目的

preprocessor展開後の解析結果を、診断、symbol、参照およびresourceごとに元の`SourceId`とrangeへ
決定的に投影する。

== 作業

* source mapを用いる単一rangeと複数segment rangeの規則を定義する。
* diagnostic、reference、resource、document symbolおよびfixのoriginを型付きで公開する。
* include directive自身とinclude先のDefinition／Document Link情報を区別する。
* nativeとWASMで同一snapshotから同一projectionを生成する。
* source map segment数と投影処理量へ上限を設定する。

== 完了条件

hostが展開後offsetを独自に推測せず、公開projectionだけから元文書へ診断とnavigationを返せる。

== 実装結果

`SourceMapping`で恒等写像と行全体への保守的写像を区別し、`origins_for_range`で複数sourceにまたがる
rangeを順序付きsegmentへ投影する。`PreprocessedAnalysis::project_origins`はdiagnostic、related
information、fix、reference、resourceおよびdocument symbolを同じsource mapから生成する。
referenceとresourceのtarget rangeは構文全体と分離し、安全に元sourceへ戻せないfixは適用不可とする。

include directiveは記述元と解決先resourceの`SourceId`を分離した。nested includeはinclude元resource
keyの親を基準に解決する。source map segment数とprojection origin segment総数には独立した上限が
あり、超過は明示的なerrorとなる。WASM API 11はmapping種別を公開し、同じsnapshotに対するnative
coreとの同値性をtestする。
