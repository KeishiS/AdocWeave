= CLIの明示的include provider
:status: completed
:depends-on: 068

== 目的

CLI hostが明示的に許可されたlocal resourceをsnapshotへ読み込み、preprocessor付き変換と検査を提供する。

== 作業

* includeを有効化する明示option、base directoryおよび許可rootを定義する。
* path traversal、symlink逸脱、循環、ファイル数、読込byteおよび個別resource sizeを制限する。
* `convert`、`check`および必要な場合の`format`に同じsnapshot構築規則を適用する。
* 標準入力と匿名sourceに対するbase URI規則を明文化する。

== 完了条件

includeは明示的に有効化した場合だけlocal I/Oを行い、同じ入力treeから決定的な結果を生成する。
network取得は対象外とする。

== 実装結果

`--include`を明示した場合だけCLI privateのlocal providerを起動する。file入力は文書の親directory、
標準入力は必須の`--base-dir`を基準とし、`--allow-root`を複数指定できる。targetは安全な相対pathに
限定し、絶対path、parent traversal、scheme、制御文字および許可root外へのsymlink解決を拒否する。

providerは`discover_includes`で依存treeを列挙し、file数、個別resource byte数および合計byte数を
制限した`ResourceSnapshot`を構築する。循環、深さ、node数および展開量はコアpreprocessorが追加で
制限する。`convert`、`check`および`symbols`は同じ展開sourceを使い、`check`は診断を元resourceへ
投影する。`format`はtreeを検証するが、展開文書ではなくroot入力だけを整形する。network I/Oは行わない。
