= Fuzzing・property test・セキュリティ試験
:status: completed
:depends-on:

== 目的

任意入力、不変条件および悪意ある入力に対するコア処理系の堅牢性を継続的に検証する。

== 実装範囲

- 任意UTF-8文字列および任意バイト列を対象に、入力境界を含めてpanicしないfuzz targetを追加する。
- パーサー、レンダラー、位置変換、属性展開、URL検証、cross reference正規化および
  Resolver境界のproperty testを追加する。
- `format(format(x)) == format(x)`、意味同値性および未変更領域のバイト保存を検証する。
- 悪意あるHTML、URL難読化、ルート外文書参照、循環・大量参照、深い入れ子、巨大行、
  閉じ忘れおよび組合せfixtureを追加する。
- 過去の不具合を最小化した回帰fixtureへ固定する運用を定義する。
- CIの短時間fuzz smoke testと定期的な長時間実行を分離する。

== 完了条件

- すべての公開入力境界にfuzz targetまたは同等のproperty testがある。
- 発見したクラッシュを再現するseed corpusをリポジトリへ保存できる。
- 上限超過とキャンセルがpanicや危険な部分HTMLを生じない。
- Formatterの三つの不変条件を自動検証できる。

== 検証

[source,console]
----
cargo test --workspace property
cargo fuzz run parser -- -max_total_time=60
cargo fuzz run render -- -max_total_time=60
----

== 先行実装

Formatter冪等性のproperty testを前倒しした。`cargo make test-properties`で実行する。
任意バイト列と長時間fuzz targetは本Issueで引き続き実装する。

== 進捗

本Issueは完了した。

`analyze`、`process`および`render`の3 targetを追加した。公開バイトAPIは不正UTF-8を含む
任意バイト列、解析とrendererは任意UTF-8を入力とする。flakeで固定したnightly Rustを
`adocweave-fuzz`にだけ使用し、通常のstable Rust環境から分離した。

決定的property suiteはCST再構築、UTF-8範囲、UTF-8・UTF-16位置往復、Formatterの冪等性・
意味保存・保護領域保存、HTML・projectionの決定性、参照キーおよびURL分類を検査する。
security suiteは属性展開後URL、不正UTF-8、処理上限、安全な構文回復、協調的キャンセル、
危険なリンク、ホストResolverが返す危険なURLおよび代表的な攻撃入力を検査する。

seed corpusと不具合最小化・回帰fixture化の手順は
xref:../docs/testing.adoc[品質保証とFuzzing]に定義した。短時間探索と長時間探索を分離し、
通常の`verify`には決定的property testとsecurity testを含める。

完了時に`cargo make verify`、`cargo make fuzz-check`および3 targetの短時間探索が成功した。
短時間探索は一時corpusを使用し、管理対象のseedを変更しないことも確認した。
