= 文書・Issue・corpus検査の持続可能な運用

== 目的

現行仕様、設計判断、完了記録および将来計画を区別し、文書corpus検査を単なる非crash確認と
規範文書の診断回帰検査へ分ける。

== 作業

* adoc-checkを非crash・range不変条件を検査するcorpus taskとして明確化する。
* 規範文書subsetに予期しない診断がないことを検査する`docs-lint`を追加する。
* 意図的な異常fixtureは期待診断を持つmanifestまたはgoldenへ分類する。
* 完了Issueを履歴として扱い、現在の規範仕様と将来ロードマップから分離する。
* 長期的な設計判断はADRへ移し、Issue作業日誌を規範仕様の正本にしない。
* Issue statusと依存関係を機械検査可能な形式にする方法を決定する。

== 完了条件

* 新しい誤診断が規範文書へ追加された場合にCIが失敗する。
* 故意の異常入力と通常文書を同じ成功基準で扱わない。
* 現行仕様、設計判断、完了履歴および未完了計画の正本が明確である。
* M12の破壊的移行を終え、workspace全体、native／WASM適合性、CLI、LSPおよびZed連携が
  新しい解析基盤だけで動作する。
* cargo make release-gateが成功し、途中移行用の旧型、adapterおよび二重pipelineが残っていない。
