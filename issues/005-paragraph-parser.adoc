= 段落の構文解析

== 目的

通常テキストと空行からlossless CSTおよび位置情報付きsemantic ASTを生成する。

== 実装範囲

- 文書、段落、テキスト、空白および改行を保持する最小CSTを定義する。
- HTML、Lint、LSPが利用する文書、段落、テキストの最小semantic ASTをCSTから構築する。
- semantic ASTにはHTMLタグ、属性、class等の出力形式固有情報を含めず、将来の別形式レンダラーも同じASTを利用できるようにする。
- 連続する非空行を一つの段落として解析する。
- 一つ以上の空行を段落の区切りとして扱う。
- CSTとASTの各要素に元入力の範囲を保持する。
- 認識できない構文候補を、生範囲と理由を持つ明示的な `Unsupported` ノードとして保持する。

== 完了条件

- 空入力、1行、複数行、複数段落を解析できる。
- 先頭と末尾の空行を安全に扱える。
- 不完全な入力でもpanicしない。
- CSTから元入力をバイト単位で復元できる。
- fixtureからCSTとASTのスナップショットを生成できる。
- パーサーcrateまたはmoduleがHTMLレンダラーへ依存していない。

== 検証

[source,console]
----
cargo test paragraph
cargo run -- convert fixtures/plain/basic.adoc
----
