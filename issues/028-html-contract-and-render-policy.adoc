= HTML出力契約とレンダーポリシー

== 目的

安全で互換性を管理できるHTML出力と、ホストが注入可能なURL・要素ポリシーを定義する。

== 実装範囲

- `render(ast, render_policy)` 相当の公開APIを定義する。
- HTMLを独立した出力バックエンドとして公開し、解析コアやsemantic ASTからHTML APIへの依存を禁止する。
- fragmentと完全HTMLのDOM構造、見出しID、class名、属性順、空白および警告の扱いを仕様化する。
- 出力可能なHTML要素、属性およびclassの許可集合を定義する。
- URLスキーム、相対URL、外部画像およびdata URIの許否を `RenderPolicy` から注入できるようにする。
- 未解決cross referenceを勝手にファイルURLへ変換せず、解決済みの安全なhrefだけを
  ホストから受け取るレンダリング境界を定義する。
- 入力由来のstyle、script、SVG、イベントハンドラおよび任意属性を生成しない。
- DOM・意味・バイト列のうち、互換性を保証する水準と契約バージョンを定義する。

== 完了条件

- 初期ノートアプリ設定で `http` と `https` だけを許可できる。
- 不正URLまたは未対応ノードの出力と診断が決定的である。
- fragmentと完全HTMLのgolden fixtureがある。
- 許可集合外の要素または属性を生成する経路がない。
- 未解決、解決失敗またはポリシー違反の参照を安全な非リンク表現と診断へ決定的に
  対応付けられる。

== 実装記録

完了。

- HTML契約バージョン1、`RenderPolicy` および `HtmlDocumentMode` を公開した。
- 既定のノートアプリ設定では `http` と `https` だけを許可し、ホストが許可
  スキームと相対URLの可否を明示的に設定できる。
- URLの大小文字、制御文字、空白、percent encodingされた制御文字、不正スキーム、
  絶対パスおよびdata URIを決定的に検査する。
- 出力要素、属性およびclassの許可集合を公開定数とテストで固定した。
- fragmentと完全HTMLのDOM構造、属性順、空白および末尾改行を
  `fixtures/html/contract.*` のgolden fixtureで固定した。
- 未解決参照はレンダラーで解決せず、安全なhrefだけをホストから受け取る境界とした。
  Issue 031で非リンク回復と位置付き診断を接続済みである。
- 詳細な契約を `docs/html-contract.adoc` に記録した。
