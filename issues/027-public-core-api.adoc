= 構文プロファイルとキャンセル可能な公開API

== 目的

CLI、サーバ、WASMおよびLSPが共有する、純粋で決定的なコアAPIを確立する。

== 実装範囲

- バージョン付き `SyntaxProfile`、`Limits`、`ParseOptions` および公開結果型を定義する。
- `parse(source, profile, limits)` 相当のAPIと、診断・CST・ASTを返す契約を定義する。
- 呼び出し側が文書を識別するための不透明な `SourceId` を任意指定できるようにし、パス、
  UUID、URL等のホスト固有表現をコアへ持ち込まない。
- 解析結果を出力形式非依存とし、HTMLその他のレンダーバックエンドを追加できる一方向の依存境界を定義する。
- 将来のcross referenceに備え、未解決の参照先表記とソース範囲を保持できるが、
  解析中にはI/Oや参照解決を行わない公開モデルを定義する。
- 呼び出し側から注入するキャンセルトークンまたは定期検査フックを追加する。
- 公開型ごとに所有権、ライフタイム、`Send`・`Sync` およびグローバル可変状態を持たない要件を決める。
- キャンセル、上限超過、入力エラーおよび内部不整合を安定したエラーコードへ対応付ける。
- 同じ入力、プロファイル、設定および処理系バージョンに対する決定性をテストする。

== 完了条件

- コアAPIがI/O、DB、時刻、乱数および環境変数へアクセスしない。
- 複数スレッドから独立した解析を安全に実行できる。
- 大きな入力の解析を呼び出し側からキャンセルできる。
- CLIとLSPが同じ公開APIを利用する。
- 文書識別子がなくても単一文書を解析でき、識別子がある場合も解析結果の意味が
  ホスト環境に依存しない。

== 実装記録

完了。

- 公開境界を `adocweave::core` に集約し、契約バージョン
  `CORE_API_VERSION = 1` を定義した。
- `SyntaxProfile`、`ParseOptions`、`ProcessingLimits`、`ParseResult`、
  `ParseError` および不透明な `SourceId` を公開した。
- `ParseResult` はlossless CST、semantic AST、決定的に整列された診断、および
  将来の参照解決に用いる未解決参照の格納先を持つ。
- `CancellationCheck`、`CancellationToken` および `NeverCancel` を定義し、
  入力走査、構文解析後、診断生成後の協調的キャンセルを実装した。
- `ParseError::code()` により、不正入力、解析失敗、上限超過、キャンセルおよび
  内部不整合を安定した `CoreErrorCode` へ対応付けた。
- CLIとLSPの文書管理を同じ公開APIへ移行した。LSPのURIは不透明な
  `SourceId` として渡し、コアはURIを解釈しない。
- 公開APIは入力文字列と明示的な設定だけを参照し、I/O、DB、時刻、乱数、
  環境変数およびグローバル可変状態を使用しない。
- 決定性、匿名ソース、不透明な文書ID、キャンセル、スレッド共有および
  `Send + Sync` の契約を単体テストで固定した。
- Issue 031で未解決参照の公開フィールドを`references`へ改め、参照モデルを
  具体化した破壊的変更に伴い、`CORE_API_VERSION`を2へ更新した。
