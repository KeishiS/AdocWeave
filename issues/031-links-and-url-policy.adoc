= リンク・cross reference構文とURLポリシー

== 目的

通常URLとAsciiDocの文書内・文書間cross referenceを解析し、外部URLと参照解決を
混同せず安全に扱う。

== 実装範囲

- URL単体、`URL[label]`、エスケープおよび属性置換の規則を定義する。
- `<<id,label>>`、`xref:#id[label]`および`xref:path.adoc#id[label]`の対応範囲、
  省略可能部分、終端規則および不完全入力からの回復を規範化する。
- リンク先と表示文字列を別のAST範囲として保持する。
- 外部URL、同一文書アンカーおよび別文書アンカーを型で区別し、対象文書表記、
  アンカー表記および表示文字列を独立した範囲として保持する。
- 相対文書表記は解析時にファイルシステムへ触れず、字句上の正規化と
  `..`、絶対パス、URL風入力等のセキュリティ方針をホスト解決前に定義する。
- スキームの大小文字、制御文字、難読化および相対URLを正規化前後で検査する。
- 不正URLに `invalid-url-scheme` 診断を付け、安全な非リンク表示へ回復する。
- 不正cross referenceと未完成cross referenceを区別する構文診断を追加する。参照先の
  存在確認はこのIssueでは行わない。
- ホバー、補完、定義移動、参照検索および文書リンクに必要な
  エディター非依存の `Reference` 情報を返す。

== 完了条件

- `http`、`https` および設定で許可したスキームだけをリンクとして生成する。
- `javascript:` 等の難読化fixtureから危険なHTMLを生成しない。
- 不完全リンクが後続インライン要素を壊さない。
- 文書内参照はIssue 030のアンカー索引に対して決定的に解決でき、未解決参照を
  位置付きで報告できる。
- 文書間参照はResolverなしでも解析、整形および参照抽出が可能である。
- 表示文字列とURLのコンテキスト別置換が規範的置換表と一致する。

== 実装記録

完了。

- URL単体、`URL[label]`、バックスラッシュエスケープおよびターゲット内の単純な
  属性置換を実装した。
- `<<id,label>>`、`xref:#id[label]`、`xref:path.adoc#id[label]` および
  `xref:scheme:locator#id[label]` を共通の `Reference` へ正規化した。
- `note:` は独立マクロではなく、`xref:note:<uuid>#id[label]` の参照先スキームとして
  解析する。
- 外部 `Link` と解決対象 `Reference` を型で分離し、ターゲット、文書、スキーム、
  locator、アンカーおよび表示文字列の独立範囲を保持する。
- `Analysis.references` がResolverなしで全参照を返す。文書内参照は同じ結果の
  `ReferenceTarget` 索引で決定的に解決する。
- 文書間表記について `..`、絶対パス、バックスラッシュ、空白・制御文字および
  URL風表記をホスト解決前に拒否する。
- URLポリシーを出力非依存の `url` モジュールへ分離し、LintとHTMLで共有する。
  大小文字、制御文字、percent encodingされた制御文字および不正スキームを検査する。
- 不正URL、不正xref、未解決の文書内xrefおよび不完全入力を位置付きで診断する。
- HTMLは安全なURLだけをリンクにし、未解決・解決失敗・ポリシー違反をエスケープ済み
  非リンク表現へ回復する。文書間・スキーム付き参照はホストが
  `render_with_resolutions()` に安全なhrefを渡した場合だけリンクにする。
- 不完全マクロをプレーンテキストとして保持し、後続インライン解析を継続する。
- 規範的な対応範囲を `docs/links-and-cross-references.adoc` に記録した。
