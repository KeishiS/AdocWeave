= 一般的なバックスラッシュエスケープ
:status: completed
:depends-on:

== 目的

リンクとxrefだけに限定したescape処理を、規範的な置換contextに基づく一般機構へ拡張する。

== 実装範囲

* 書式、属性参照、URL自動認識、anchorおよび対応済みmacroに対するescape規則を定義する。
* lossless CSTでバックスラッシュと対象範囲を保持する。
* semantic ASTでは表示文字とescape由来のliteral性を、backendによる再解析なしで表現する。
* normal、verbatimおよびliteral contextごとの適用可否を置換表へ追加する。
* Formatter、診断、HTML、LSP semantic tokenおよびHoverを同じ意味モデルへ同期する。

== 完了条件

* `\*text*`、`\{name}`、`\https://example.org`およびescaped macroが文字どおり表示される。
* リテラル、ソース、等幅およびSTEMの不透明内容を誤って変更しない。
* 末尾の単独バックスラッシュと連続バックスラッシュが決定的に回復する。
* 規範文法、置換表、互換性表およびconformance fixtureが更新される。
* `cargo make release-check`が成功する。

== 依存関係

* 047

== 進捗

normal contextのconstrained書式、属性参照、URL、xref、STEM macroおよびanchor開始記号へ
共通のescape処理を追加した。奇数個の連続バックスラッシュは最後の一つで構文認識を抑止し、
偶数個では認識を維持する。末尾バックスラッシュ、不完全escaped macro、opaqueな等幅・STEM内容、
Unicode安全なrangeおよび診断抑止を回帰試験で固定した。

規範文法、置換表、互換性表、対応状況表およびnative／WASM共通conformance fixtureを更新した。

既定設定の`cargo make release-check`は2026-07-20に成功した。Core 160 tests、実WASMとの
10件の適合性比較、3 target各5秒のfuzz smokeを含む。本Issueは完了した。
