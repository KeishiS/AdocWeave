= 文書表示属性：TOCとsection番号
:status: planned
:depends-on: 074,112

== 目的

既存の`DocumentStructure` projectionを用い、`toc`、`toclevels`および`sectnums`を文書表示属性として
解決し、HTML出力を制御する。

== 作業

* document attributeからTOCの有効化、表示深さ、section番号の有効化を型付きpresentation policyへ解決する。
* article、book、manpage、appendix、part、discrete headingに対するTOC・番号の適用範囲を明示する。
* HTMLが既存TOC entryとsection treeを再利用し、指定した深さと番号規則で出力するようにする。
* document symbols、xref ID、LSP hoverおよびJSON projectionが表示用番号の変更で不安定にならないことを保証する。
* 公開契約への影響を判定し、必要ならversion、fixtureおよびRelease noteを同期する。

== 検証

* `toc`、`toclevels`、`sectnums`、未指定値、無効値、article／book／manpage、appendix、part、discrete headingをfixture化する。
* nativeとWASMのHTML、TOC projection、LSP document symbolsおよびxref解決が同じsection treeを共有することを確認する。

== 完了条件

TOCとsection番号が一つのpresentation policyと既存`DocumentStructure`から決定され、source ID・reference解決・semantic ASTを変更しない。

== 範囲外

theme固有のTOC配置、CSSの見た目、DocBook／PDF固有の番号設定、任意の出力backendは扱わない。
