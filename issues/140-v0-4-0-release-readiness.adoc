= v0.4.0 release readinessとstable直送
:status: in-progress
:depends-on: 136, 137, 138

== 目的

typed admonition、quote／verse presentationおよび対応構文文書のmachine-readableな同期を、公開契約version 3の
破壊的更新として`v0.4.0`へまとめる。個人開発の実際の受け入れ経路に合わせ、release candidateを廃止し、
mainで成功したcandidateをstable tag一回で公開する。

== 設計判断

release manifestをrelease trainの正本とし、workspace、Zed、browser、distribution planとfixtureの値は
`cargo make release-contract`で一致を検証する。Cargoとnpmのpackage metadata、およびcargo-distが読む静的な
extra artifact名には同じversionを持つ必要があるため、これらを生成時に書き換える隠れた状態は導入しない。
版数更新は一つのreview可能なcommitとし、欠落は自動gateで拒否する。

RC tag、prerelease分岐および二段階のversion同期は廃止する。公開はmainの成功commitに`vX.Y.Z`を付ける一回だけの
操作とし、既存releaseは置換しない。公開後の不具合は新しいstable versionで修正する。

== 完了条件

* workspace、Zed、browser、release manifest、distribution planおよびfixtureが`0.4.0`で一致する。
* 公開tagは`vX.Y.Z`だけを受理し、workflowは常にstable Releaseだけを作成する。
* `docs/upgrade-0-4.adoc`が契約version 3への移行を定義する。
* `cargo make release-gate`と`node tools/release-contract.mjs --tag=v0.4.0`が成功する。
* mainのcandidate成功後、`v0.4.0`を作成してGitHub Actionsと公開assetを受け入れる。

== 実施結果

版数、lockfile、配布計画およびfixtureを`0.4.0`へ同期し、RC専用のtag、schemaおよびpublish分岐を削除した。
`cargo make release-gate`および`node tools/release-contract.mjs --tag=v0.4.0`は2026-07-23に成功した。
残る作業は、このcommitをmainへpushしたcandidateの受け入れとstable tagの公開だけである。

== 公開前review是正

公開準備後のsemantic block reviewで、AsciiDoc cell内のblock presentation、list continuation内の
paragraph admonition、および散文中の`TODO:`形式のURL誤認が見つかった。release前に次を是正する。

* `walk_blocks_mut`を使い、table cell、compound blockおよびlist continuationを含む全block到達経路で
  delimited presentationを解決する。
* continuation paragraphをroot paragraphと同じ意味構築関数へ通し、admonitionとinline診断を一致させる。
* 自動URL認識を`//`を持つURL、`mailto:`、または同一tokenの明示label付きlinkへ限定する。
* admonition label rangeから後続空白を除外し、styleなしquoteとverseのAsciidoctorとの差異を互換性表へ記録する。

これらはtable cell内のadmonition・quote・verse、continuation admonition、散文colonおよびrangeの回帰テストで固定する。
