= 標準macroとresource境界

AdocWeaveは標準macroを、macroごとの文字列再走査ではなく、共通の`StandardMacro`として表現する。
各nodeはkind、inline／block form、全体range、targetとtarget range、属性listとrange、および属性展開結果を
持つ。`Analysis.macros()`が全macroを、`Analysis.resources()`と`resource_queries()`がmedia参照だけを返す。

== 対応するmacro

次を認識する。

* bare email addressと`mailto:` link
* `footnote:[text]`および`footnote:id[text]`
* `anchor:id[]`、inlineの`[[id]]`、`bibanchor:id[]`および`[[[id]]]`
* `indexterm:[primary,secondary]`
* `kbd:[keys]`、`btn:[label]`および`menu:Menu[Item,Subitem]`
* inline／blockの`image`、`icon`、`audio`および`video`

inline anchorとbibliography anchorは共通の参照先索引へ追加する。さらに、解析時に一度だけ構築する
backend非依存の文書catalogへfootnote、bibliography entry／参照edgeおよび階層index termを保持する。
index termは表示テキストおよび検索テキストへ混在させない。

`footnote:[text]`は匿名定義、`footnote:id[text]`は再利用可能な名前付き定義、`footnote:id[]`は
名前付き定義への参照である。番号は最初の出現順に決まり、HTML backendはinline位置へ番号付き参照を、
文書末尾へback reference付き一覧を生成する。前方参照を許可し、欠落定義と重複定義は元rangeを持つ
`invalid-catalog`診断になる。
名前付きfootnote参照は別IDへのaliasを定義できないため、現在の型では循環参照を構築できない。
将来alias構文を追加する場合は、catalog edgeとcycle診断を同じ契約変更で追加する。

`bibanchor:id[]`はbibliography entryとなり、同じIDへのlocal xrefを参照edgeとしてcatalogへ接続する。
`indexterm:[primary,secondary,...]`は空でない階層pathとして正規化し、同じpathの出現rangeをまとめる。
重複bibliography IDと空index termも`invalid-catalog`である。catalogは`AstDocument::catalogs()`、
`Analysis::catalogs()`およびJSON projectionから取得でき、consumerがASTを再走査して再構築しない。

UI macroは`RenderPolicy.render_ui_macros`を明示的に有効化した場合だけ専用HTMLを生成する。無効時も
表示文字列を失わない。これは構文認識を切り替えるflagではなく、backend presentationのfeature flagである。

== resource resolver

画像、icon、音声および動画は`ResourceReference`へ射影する。コア解析は存在確認、file／network取得、
data URI化、MIME判定または権限判定を行わない。ホストは`ResourceResolver`へ同じsnapshot由来の
`ResourceQuery`を渡し、`ResourceValue`または型付き失敗を返す。結果は元macro全体のrangeと
`ResourceOutcome`を持つ`ResolvedResource`へ変換し、reference結果とともに`RenderInputs`へ格納する。

resolverが返したhrefも信頼済みではない。HTML rendererは`UrlPolicy`で再検査する。解決結果が欠落、
失敗、重複または不許可の場合はactive resource要素を生成せず、安全なfallbackと位置付き診断を返す。
widthとheightは
10進整数だけを許可し、入力由来の任意属性、event handler、raw HTMLを通過させない。

== 資源上限

catalog entry数は重複occurrenceを含め`max_catalog_entries`、文字列とrange投影量は
`max_catalog_bytes`で構築時に制限する。HTML rendererはcatalogを参照するだけでresolverやI/Oを呼ばない。

== 今後の拡張

catalogと`[bibliography]` sectionを結ぶ出力は対応済みである。citation styleの自動整形、外部citation database、
index range marker、
see／see-also関係、icon set解決、poster／start属性、YouTube／Vimeo video target、
新しいmacroは`StandardMacroKind`、共通scanner、
walker、projection、HTML allowlist、resource queryおよびconformance表現を同じ変更で更新する。
