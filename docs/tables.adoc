= 表の対応仕様

AdocWeaveはAsciiDocの表を、区切り付きblockの文字列ではなく、入力形式、separator、列、行、cell、
span、配置、style、内容modelおよびsource rangeを持つ意味モデルとして扱う。sourceの基準はlossless syntax treeであり、
意味モデルからsourceを再生成しない。

== 対応範囲

次を対応する。

* `|===`によるPSV表
* `format="csv"`、`format="dsv"`、`format="tsv"`による区切り形式
* `separator`による1文字separatorの上書き
* CSV／DSV／TSVのquoted cell、二重quote escapeおよび複数行cell
* 先頭行のcell数による暗黙列数と、`cols`による明示列数
* `cols`の幅、`n*` multiplier、水平・垂直配置およびcell style
* `options="header,footer"`による先頭・末尾行の区分
* block titleによるtable caption
* `frame`、`grid`、`stripes`の固定表示class、および`width`の1から100までの百分率
  （`%`の有無は問わない）
* `options="autowidth"`。`width`との同時指定は診断し、autowidthを優先する
* `2+|`、`.2+|`、`2.3+|`によるcolumn span、row spanおよび両方のspan
* cell固有の水平・垂直配置と、default、emphasis、header、literal、monospace、strong、
  verse、AsciiDocのstyle
* 通常cell内のinline置換と複数行内容
* `n*|value`によるPSV cell duplication
* AsciiDoc style cell内の対応済みblock sequence全体の再帰解析
* `!===`等の別separator delimiterによるAsciiDoc cell内の入れ子表
* 表本体byte数、cell数、列数および表深さの構築時上限

列指定よりcell指定を優先する。header行は列またはcellのstyleにかかわらずheader要素として
出力する。HTML backendは`table`、section、row、cell要素と、契約で許可した属性・classだけを
生成し、入力由来のHTMLを通過させない。

== 形式とescape

PSVはseparator直前の奇数個のbackslashによってcell境界の認識を抑止する。CSV、DSVおよびTSVは
double quoteで囲んだcellを認識し、quoted cell中の`""`を1個の`"`として意味値へ復号する。
quoted cell中ではseparatorと改行を内容として保持する。DSVの既定separatorは`:`、TSVはtabである。
入力形式は`TableInputSpec::resolve`だけが決定する。標準`|===`では明示`format`と`separator`を使用する。
custom delimiterでは先頭文字がseparatorの基準であり、`,`と`:`は明示`format`がない場合にCSVとDSVを
推論する。明示`format`はこの推論を上書きするが、custom delimiterと異なる`separator`属性は競合として
`invalid-table`診断になる。

custom separatorは空白、制御文字、`=`以外のUnicode scalar valueをちょうど1個指定する。未知の`format`、
空・複数文字・禁止文字の`separator`、閉じていないquoted cellも`invalid-table`診断になる。

== AsciiDoc style cell

`a` style cellはinline列ではなく、独立したblock列として解析する。文書root、compound containerおよび
AsciiDoc cellは、context policyだけを切り替える単一のblock sequence状態機械を使う。見出し、paragraph、
list、literal paragraph、break、source／STEM、metadata付きblock、区切り付きblockおよび入れ子表を
同じrecognizer／builderで構築する。子blockはHTML、検索projection、lintとsemantic walkerから到達できる。
cell解析は親解析とnode、block、表深さ、cell数およびcancel checkを共有する。
PSVでcell内容が元source範囲と一致する場合は、root source storageと既存行境界から作るindexed viewを使用し、
cellごとの文字列複製と改行再走査を行わない。quote解除等により内容が変換されたcellだけが明示的な変換済み入力を
構築する。

== 現在の制限

captionはblock titleと同じ解決済みinline列として出力する。自動header推論は未対応である。CSV系の
quoted cellを`a` column styleにした場合、復号後文字列と元sourceが一対一にならないため、現段階では
block解析を保証する対象をPSVのAsciiDoc style cellに限定する。未知の形式を暗黙に別形式として
成功扱いにはせず、診断を返した上で安全なPSV fallbackを保持する。

document title、著者、改訂およびdocument attributeは文書rootだけの状態であり、AsciiDoc cellでは開始しない。
cell内の見出しはsectionまたはdiscrete headingとして扱い、文書全体のTOC階層には加えない。明示anchor、
診断および参照targetは文書全体の基準へ統合する。

== 置換と安全性

通常cellは`Normal`、literalとverseは`Verbatim`、AsciiDoc style cellは子blockごとの置換contextを
適用する。HTMLへの変換時には文字列と属性値を出力文脈ごとにescapeする。
PSV separatorのbackslash escapeはcell境界の走査時に認識し、inline置換へsource rangeを保ったまま
渡す。
