= cross reference解決契約

コア解析は参照解決を行わない。`ParseResult.references`は範囲付きの構文表現を保持し、
`reference_queries()`は別文書、`note:`およびその他のスキーム参照を
`ReferenceQuery`へ正規化する。参照元は不透明な`SourceId`のままホストへ渡す。

`note:`は独立したマクロではなく、xref参照先スキームである。

[source,asciidoc]
----
xref:note:123e4567-e89b-12d3-a456-426614174000[ノート]
xref:note:123e4567-e89b-12d3-a456-426614174000#section[節]
----

UUIDは小文字のcanonical形式だけを受理し、不正値には`invalid-note-uuid`を付ける。
アンカーは任意である。入力途中の閉じていない表示文字列はテキストとして回復する。

`ReferenceResolver`は文書候補、相対文書、UUID、アンカーおよび逆参照を非同期に
問い合わせるホスト境界である。コアはDB、ファイルシステム、ワークスペースルートを
知らない。逆参照結果は`SourceId`と参照元`TextRange`を保持する。

失敗は対象不在、アンカー不在、曖昧、ルート外およびResolver障害に分け、それぞれ
安定した診断コードへ対応する。キャッシュキーは参照元、正規化した参照先、
構文プロファイル版および文書versionから構成する。

Resolverが返したhrefは信頼境界を越える外部入力である。HTMLは必ずURLポリシーを
再適用し、拒否したhrefをリンクとして出力しない。
