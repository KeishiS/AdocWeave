= 文書属性

== 適用範囲

文書タイトルの直後に空行を挟まず記述された属性行を文書属性として解析する。
本文中やセクション見出し後の同じ表記は文書属性として扱わない。

[source,asciidoc]
----
= ノート
:note-id: 123e4567-e89b-12d3-a456-426614174000
:tags: rust, AsciiDoc
:stem: latexmath

本文
----

定義は `:name: value`、解除は `:name!:` または `:!name:` とする。属性名に使用できる
文字はASCII英数字、`_`、`-` および `.` である。CSTは属性行の空白、大小文字、
改行および解除形式を元のバイト列のまま保持する。

== 属性参照

通常のインライン文脈では `{name}` を単純な属性参照として解析する。

[source,asciidoc]
----
:project: AdocWeave

これは {project} の文書です。
----

展開は一回だけ行う。属性値に含まれる別の `{name}` は再帰展開しない。ソースコード、
リテラルおよび等幅文字の内部では属性置換を行わない。未定義参照は原表記を安全に
出力し、`undefined-attribute` 診断を生成する。

== 型付きノートメタデータ

[cols="1,2"]
|===
|属性 |値

|`note-id`, `creator-id`
|ハイフンを含む36文字のUUID。ASTではASCII小文字へ正規化する。

|`created-at`, `updated-at`
|タイムゾーンを含むRFC 3339形式の日時表記。

|`tags`
|カンマ区切り。各要素の前後の空白を除去し、順序と大小文字を保持する。

|`stem`
|現時点では `latexmath` または `latex`。ASTでは `LatexMath` に正規化する。
|===

上記以外の属性値は文字列として保持する。不正な型付き値は原文を失わず
`invalid-attribute` として診断する。

== 保護属性

ホストは `ParseOptions.protected_attributes` に権威的な属性値を渡せる。ソース中の
定義または解除がその値を変更する場合、寛容モードでは警告、厳格モードではエラーの
`protected-attribute` 診断を生成する。コアはDBや環境変数から値を取得しない。

== 対象外

- 再帰的な属性展開
- 条件分岐
- 環境変数または外部コマンド参照
- 属性値に基づくinclude
- 本文途中での文書属性変更
