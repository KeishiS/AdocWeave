= 文書属性

== 適用範囲

文書タイトルの直後に空行を挟まず記述された属性行を文書属性として解析する。
本文中やセクション見出し後の同じ表記は文書属性として扱わない。

[source,asciidoc]
----
= 文書
:project: AdocWeave
:stem: latexmath

本文
----

定義は`:name: value`、解除は`:name!:`または`:!name:`とする。属性名に使用できる
文字はASCII英数字、`_`、`-`および`.`である。CSTは空白、大小文字、改行および
解除形式を元のバイト列のまま保持する。

semantic ASTの`DocumentAttribute`は、名前、原値、定義または解除の操作と各ソース範囲を
保持する。標準コアは属性名に応じたUUID、日時、タグ等への型変換を行わない。

== 属性参照

通常のインライン文脈では`{name}`を単純な属性参照として解析する。展開は一回だけ行い、
属性値に含まれる別の参照を再帰展開しない。ソース、リテラル、等幅文字および数式の内部
では属性置換を行わない。未定義参照は原表記を保持し、`undefined-attribute`診断を生成する。

`stem`は標準STEM構文の解釈時に意味を持つが、汎用属性AST自体では文字列として保持する。

== 保護属性

ホストは`ParseOptions.protected_attributes`へ権威的な属性値を渡せる。ソース中の定義
または解除がその値を変更する場合、寛容モードでは警告、厳格モードではエラーの
`protected-attribute`診断を生成する。コアは値をDBや環境から取得しない。

== アプリケーション固有メタデータ

`note-id`、`creator-id`、日時、タグ等の名前、型、必須性および正規化規則は標準AsciiDoc
の属性仕様ではない。これらは外部ホストが`Analysis.ast.attributes`から検証・投影する。
規範契約はxref:extensions.adoc[ホスト拡張実装契約]を参照する。

== 対象外

* 再帰的な属性展開
* 条件分岐
* 環境変数または外部コマンド参照
* 属性値に基づくinclude
* 本文途中での文書属性変更
