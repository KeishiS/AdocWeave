= v0.5.0への移行

v0.5.0はblock title／table captionを解決済みinline presentationへ移行し、quote-aware include属性と
明示`optional` includeを追加する。またホスト供給stylesheetをcomplete document出力へ適用する
`RenderPolicy.stylesheets`、CLIの`--complete`／`--css`／`--css-url`およびWASMの`documentMode`／
`stylesheets`設定を追加する。公開契約は3から5へ更新する。

== 必要環境と契約値

sourceからv0.5.0をbuildするRust hostはRust 1.97.1以上を使用する。binary releaseだけを利用する場合は
Rust toolchainを必要としない。package version、Rust versionおよびcontract versionの機械可読な正本は
xref:../release-manifest.json[v0.5.0 release manifest]である。hostはpinned contractを3から5へ更新する。

== 破壊的変更

* `BlockMetadata.title`はraw metadata valueではなく、source値、range、解決済みinline列を持つ`BlockTitle`である。
* table captionとadmonition titleはliteral textではなく、属性展開・inline書式・安全なmacro出力を適用する。
* `PreprocessedDocument`はnon-fatalな`notices`を公開し、missing resourceの`optional` includeを
  `optional-resource-missing`として報告する。
* core crateの`Operation`、`ProcessError`、`ProcessConfig`、`process`および`process_check`を削除する。
  hostは`Engine::analyze`の結果を必要なrenderer、formatterまたはdiagnostic serializerへ明示的に渡す。
* Rustの公開moduleを`semantic`、`output`、`preprocess`、`resolution`および`text` facadeへ再編する。
  旧`parser`、`html`、`projection`等のroot module pathは提供しない。
* table captionの正本を`BlockMetadata.title`だけに限定し、`TablePresentation.caption`を削除する。
* `RenderPolicy`へ`stylesheets`フィールドが追加される。網羅的なstruct literalで構築するhostは
  更新が必要である。stylesheetを設定しない既定のHTML出力は従来とbyte単位で一致する。
* HTML契約の許可集合へ`head`、`style`、`link`が加わる。許可集合を検証器へ固定しているhostは更新する。
* WASM、browser、CLI、LSPおよびhost adapterはcontract version 5だけを受理する。

== Rust公開APIの移行

次の表は、v0.4.xまでcrate rootから公開していたmodule pathとv0.5.0の置換先を示す。`pub use`で
旧pathを残していないため、importを新pathへ一括移行する。

[cols="2,2,3",options="header"]
|===
|v0.4.xのpath |v0.5.0のpath |用途

|`adocweave::parser`、`inline`、`catalog`、`document`、`presentation`、`structure`、`substitution`、`table`、`walker`
|`adocweave::semantic`
|semantic AST、文書構造、catalogおよび走査

|`adocweave::html`、`conformance`、`diagnostic`、`lint`、`formatter`、`projection`
|`adocweave::output::{html, conformance, diagnostics, formatter, projection}`
|個別の出力backend

|`adocweave::preprocessor`
|`adocweave::preprocess`
|I/Oを行わないinclude前処理とsource map

|`adocweave::reference`、`render`、`resource`、`url`
|`adocweave::resolution`
|host供給の参照・resource解決結果とrenderer入力

|`adocweave::source`、`syntax`
|`adocweave::text`
|source位置、source documentおよびlossless syntax tree

|`adocweave::{Operation, CheckOutput, ProcessError, process, process_with_config, process_check, process_check_with_config}`
|削除。`Engine::analyze`と用途別output APIを組み合わせる
|CLI向け一括処理APIはhostまたはCLIが所有する
|===

`DocumentAttribute`と`AttributeOperation`は、source属性の構築途中だけに必要な内部型として非公開化した。
最終的な属性値は`analysis.presentation().attributes()`から`ResolvedDocumentAttributes`として読む。source順、
set／unset操作またはrangeが必要なconsumerには、v0.5.0では安定した公開APIを提供しない。
v0.6.0以後はxref:upgrade-0-6.adoc[公開occurrence query]を使用する。この内部型や`AstDocument`の
内部fieldへ依存した公開APIは作らない。

== 更新手順

. hostの`CONTRACT_VERSION`照合値を5へ更新する。
. Rust hostのimportを用途別facadeへ移し、CLI相当の処理は`Engine`と各出力APIから構成する。
. block titleをraw HTMLとして扱う後処理を削除し、AdocWeaveのHTMLまたはprojectionをそのまま利用する。
. includeの不在を許容する必要がある箇所だけ`optional`を明示し、noticeを利用者へ表示するか記録する。
. 同じ`v0.5.0` ReleaseのCLI、LSP、browser archiveおよびZed archiveを同時に導入する。
