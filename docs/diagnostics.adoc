= 診断契約

標準診断は構文・意味解析から生成し、`Analysis.diagnostics`へ同じ文書版の結果として
格納する。CLI、LSP、HTMLその他のconsumerは独自にLintを再実行しない。

Issue 055では生成経路を、syntax認識時の`SyntaxDiagnostic`、semantic lowering時の診断、
意味・style lintに分ける。回復済みASTに保持した`problems`をLintが再び標準診断へ変換する
重複を廃止し、`Analysis`構築時に一度だけ安定順序へ統合する。

== 安定識別子

`LintRule`を標準診断コードの単一レジストリとする。診断コードはASCII小文字、数字、
`-`から構成し、利用者向けmessageではなくcodeを機械判定に用いる。

診断IDはcodeと範囲から決定的に生成する。同じ入力、設定およびコアversionからは、
範囲、severity、message、関連情報およびFixを含め同じ順序の診断を返す。

== 構文診断とホスト診断

標準コアはAsciiDoc構文、対応範囲、URLポリシーおよび明示的な保護属性だけを診断する。
ノートID、DB、権限、外部文書の存在等は解析後のホスト診断である。ホスト診断は
`Analysis`を書き換えず別の結果として生成し、表示時に安定順序で統合する。

== Fix

Fixは重複しない`TextEdit`だけを保持する。`Always`は構文上安全に適用できる変更、
`Maybe`は利用者判断を必要とする変更を表す。FormatterとCode Actionは同じEdit契約を
利用する。

== Language Serverへの公開

Language Serverは同じ`Analysis`に属する診断と`LineIndex`を使用してLSP範囲へ変換する。
文書更新後に完了した古い解析、またはキャンセル済み解析の診断を公開してはならない。
通知順序と結果採用規則はxref:language-server.adoc[Language Server設計]に従う。
