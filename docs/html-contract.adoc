= HTML出力の仕様

このページは、生成されるHTMLをプログラムで利用する人向けの詳細です。通常は`adocweave convert`の
出力をそのまま使えます。HTMLの見た目はxref:browser-library.adoc[ブラウザーで使う]と
`RenderPolicy.stylesheets`の説明を参照してください。

== Versionと互換性

HTML出力は独立した契約versionを持たない。生成元のpackage versionはxref:../release-manifest.json[release manifest]、
WASM応答およびprojection JSONの`packageVersion`で識別する。HTML DOM自体へversion値は埋め込まない。

互換性はDOM構造と意味について保証する。属性順、改行および空白もgolden fixtureで固定するが、将来の
package versionでは変更できる。許可集合、見出しID規則またはDOMの意味を変更する場合は、fixtureと利用側の
検証器を同じpackage version更新で確認する。異なるpackage version間のバイト列互換性は保証しない。

== 文書モード

`Fragment` は本文要素だけを出力する。既定値は `Fragment` である。

`Complete` は次の固定構造で囲む。

[source,html]
----
<!doctype html>
<html>
<body>
...
</body>
</html>
----

stylesheetを1件以上出力する場合だけ、`<html>`の直後に`<head>`を挿入する。

[source,html]
----
<!doctype html>
<html>
<head>
<style>
...
</style>
<link rel="stylesheet" href="...">
</head>
<body>
...
</body>
</html>
----

== ホスト供給stylesheet

`RenderPolicy.stylesheets`は、ホストまたはCLIが明示的に渡す出力設定である。文書入力からCSSへ
到達する経路はなく、入力由来の`style`属性・任意class・raw HTMLの拒否は変更しない。設計判断は
xref:adr/0007-host-supplied-stylesheets.adoc[ADR 0007]に記録する。

* sourceは埋め込みCSS本文（`<style>`）と外部URL（`<link rel="stylesheet">`）の二種類で、
  `Complete`モードの`<head>`へホスト指定順に出力する。同一内容のsourceは最初の一つだけを出力する。
* `Fragment`モードではCSSを出力せず、設定されていれば`stylesheet-not-applicable`診断を返す。
* 埋め込みCSS本文はHTML escapeで保護できないため、`</style`（ASCII大小文字非区別）、`<!--`および
  改行・タブ以外の制御文字を含む本文を拒否し、`invalid-stylesheet-content`診断を返す。
* stylesheet URLは`UrlPolicy`を`ResolvedResource` contextで検査し、href属性値としてHTML escapeして
  出力する。拒否時は`invalid-stylesheet-url`診断を返し、`<link>`を生成しない。
* 既定上限は埋め込みCSS本文1 sourceあたり1 MiB、URL 2,048 byte、source 16件である。超過分は
  `stylesheet-limit-exceeded`診断とともに出力しない。
* stylesheet診断は文書先頭の空rangeを持ち、CSS本文やURLの値を診断messageへ含めない。

== 許可集合

公開baseline 1のレンダラーが生成できる要素は次に限定する。

- `html`
- `head`、`style`、`link`（ホスト供給stylesheetを設定したcomplete documentだけ）
- `body`
- `a`
- `br`
- `h1` から `h5`
- `p`
- `div`
- `strong`
- `em`
- `code`
- `pre`
- `ul`
- `ol`
- `li`
- `hr`
- `dl`、`dt`、`dd`
- `mark`、`span`、`sub`、`sup`
- `table`、`caption`、`thead`、`tbody`、`tfoot`、`tr`、`th`、`td`
- `img`、`audio`、`video`、`kbd`、`blockquote`、`cite`

生成できる属性は `id`、`class`、`href`、`colspan`、`rowspan`、`src`、`alt`、`width`、
`height`、`title`、`controls`、`target`および`rel`に限定する。`target`と`rel`は
`ExternalLinkPresentation`が外部HTTP(S)リンクへ固定値を生成する場合と、ホスト供給stylesheetの
`<link rel="stylesheet">`だけに使用し、入力由来属性を通過させない。classは
実装の`ALLOWED_CLASSES`に列挙した文書metadata、list、数式、page break、表配置、admonition、quoteおよびverse用の固定値に
限定する。入力由来の任意属性、`style`、
イベントハンドラ、`script`、SVGおよび生HTMLは生成しない。

許可集合を検証器やサニタイザーへ固定しているホストは、package version変更時に更新が必要である。

== URLポリシー

URLは `RenderPolicy` により検査する。既定では `http` と `https` だけを許可し、
相対URL、data URIおよび外部画像を許可しない。ホストは許可スキームを明示的に
追加できる。

判定には`AuthoredLink`、`ResolvedReference`または`ResolvedResource`の由来contextを必ず渡す。
`allowResolvedRootRelative`はhost解決値だけに適用し、文書へ直接記述されたroot-relative URLを許可しない。
`//host/path`、path traversalおよびbackslashは解決値でも拒否する。

スキームはASCII大小文字を区別せず正規化する。制御文字、空白、percent encoding
された制御文字、不正なスキーム、絶対パスおよびバックスラッシュで始まる相対表記を
拒否する。

解決済みxrefは、明示label、Resolverが渡したプレーンテキスト`display_text`、元targetの順で表示する。
明示labelだけは通常のinlineとして描画し、`display_text`はinlineやHTMLとして再解析せずHTMLエスケープする。
未解決またはポリシー違反の参照はリンクへ変換せず、表示文字列をHTMLエスケープした
非リンク表現として出力し、位置付き診断を返す。media resourceも同様に、解決済みURLがない場合は
画像のaltまたはmedia targetを安全なテキストとして出力する。参照・resource解決はレンダラー内で行わない。
入力契約はxref:render-inputs.adoc[解決済みrenderer入力契約]を参照する。

`SourceLanguagePolicy`はsource blockの言語allowlistと未知言語時のclass省略・診断を制御する。
`MathLanguagePolicy`は許可した数式言語だけへ数式classを付ける。`ResourceCapabilities`で画像またはmediaを
無効化した場合、解決結果が存在してもactive要素を生成しない。

== 属性順と空白

- 見出しでは `class` を `id` より前に出力する。
- 一要素を一行に出力する。
- fragment、complete documentとも末尾改行を付ける。
- 段落内の連続するソース行は単一のASCII空白で結合する。
