= HTML出力契約

== 契約バージョン

HTML出力契約の現在のバージョンは `7` であり、`HtmlOutput.contract_version` に
記録する。契約バージョンを変更する場合は、
golden fixture、許可集合および見出しID規則の差分を確認する。

バージョン7ではPSV表のため、`table`、`thead`、`tbody`、`tfoot`、`tr`、`th`、`td`、
`colspan`、`rowspan`および固定の配置classを許可集合へ追加した。

互換性はDOM構造と意味について保証する。属性順、改行および空白もgolden fixtureで
固定するが、将来の契約バージョンでは変更できる。異なる契約バージョン間の
バイト列互換性は保証しない。

== 文書モード

`Fragment` は本文要素だけを出力する。既定値は `Fragment` である。

`Complete` は次の固定構造で囲む。

[source,html]
----
<!doctype html>
<html>
<body>
...
</body>
</html>
----

== 許可集合

バージョン7のレンダラーが生成できる要素は次に限定する。

- `html`
- `body`
- `a`
- `br`
- `h1` から `h5`
- `p`
- `strong`
- `em`
- `code`
- `pre`
- `ul`
- `ol`
- `li`
- `hr`
- `dl`、`dt`、`dd`
- `mark`、`span`、`sub`、`sup`
- `table`、`thead`、`tbody`、`tfoot`、`tr`、`th`、`td`

生成できる属性は `id`、`class`、`href`、`colspan`および`rowspan`に限定する。classは
実装の`ALLOWED_CLASSES`に列挙した文書metadata、list、数式、page breakおよび表配置用の固定値に
限定する。入力由来の任意属性、`style`、
イベントハンドラ、`script`、SVGおよび生HTMLは生成しない。

許可集合を検証器やサニタイザーへ固定しているホストは、契約version変更時に更新が必要である。

== URLポリシー

URLは `RenderPolicy` により検査する。既定では `http` と `https` だけを許可し、
相対URL、data URIおよび外部画像を許可しない。ホストは許可スキームを明示的に
追加できる。

スキームはASCII大小文字を区別せず正規化する。制御文字、空白、percent encoding
された制御文字、不正なスキーム、絶対パスおよびバックスラッシュで始まる相対表記を
拒否する。

未解決またはポリシー違反の参照はリンクへ変換せず、表示文字列をHTMLエスケープした
非リンク表現として出力し、位置付き診断を返す。参照解決はレンダラー内で行わない。

== 属性順と空白

- 見出しでは `class` を `id` より前に出力する。
- 一要素を一行に出力する。
- fragment、complete documentとも末尾改行を付ける。
- 段落内の連続するソース行は単一のASCII空白で結合する。
