= 対応範囲の定義

本書は、AdocWeaveが環境に依存せず扱う範囲を定義する。通常の文書作成では
xref:syntax-support.adoc[対応しているAsciiDocの書式]を参照。

== 何を含むか

AdocWeaveは、AsciiDocの読み取り、問題の検出、整形、HTMLへの変換、見出しやリンクなどの構造情報を提供します。
特定のノートアプリのデータベース、ログイン権限、画面の構成、ネットワーク通信は提供しません。
そのような機能は利用側アプリで追加します。

== バージョン

公開API、WebAssembly、HTML出力、構造情報は、同じパッケージバージョンを使います。現在の`VERSION = 0.6.0`です。
現在の値と必要なRustのバージョンはxref:../release-manifest.json[release manifest]が基準です。
公開結果を保存するアプリケーションは、AdocWeaveの更新時にHTMLおよび検索用情報を再構築すること。

0.y.zの間はminor版で互換性のない変更を行う場合がある。更新前に
xref:current-contract.adoc[公開している仕様と移行情報]の該当する移行ガイドを確認すること。

== 対応する書式

文書の題名、見出し、段落、属性、アンカー、リテラル・ソースブロック、リスト、表、リンク、
相互参照、数式、画像・音声・動画などを扱います。各書式の対応状況は
xref:syntax-support.adoc[対応しているAsciiDocの書式]にあります。

未対応の書式を通常のHTMLとしてそのまま出力することはありません。`SyntaxMode::Permissive`では
内容を安全なテキストとして残し、`SyntaxMode::Strict`では未対応の書式を含む文書を処理しません。

== 処理の上限

大きすぎる入力で処理が止まらないように、既定の上限を設けています。

[cols="2,1",options="header"]
|===
|対象 |既定の上限

|入力 |10 MiB
|出力 |50 MiB
|1行 |1 MiB
|リストの深さ |8
|インライン書式の深さ |32
|表の本体 |5 MiB
|表のセル |100,000
|表の列 |1,000
|ブロック |100,000
|問題の件数 |1,000
|===

利用側アプリはより小さい上限を指定できる。CLIとWebAssemblyは出力サイズも検査する。
ライブラリを直接使用してHTMLを表示するアプリは、表示前に同等の出力サイズ検査を行うこと。

== API利用

Rustからは`Engine`、`ParseOptions`、`Analysis`が主な入口です。属性の最終値は
`Analysis::presentation().attributes()`から、属性を記述した順番や位置は
`Analysis::document_attribute_occurrences()`から取得する。

HTML、整形、問題一覧、構造情報、外部リンクの解決結果は用途ごとの公開モジュールにあります。
型名およびJSONの詳細は、各ページからソースコードのAPIドキュメントを参照。
