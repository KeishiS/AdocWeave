= AdocWeave Core Profile 2

この文書は、ホスト非依存な標準構文profile 2の公開契約、互換性判断および再構築条件を
定義する。特定ノートアプリのメタデータ、DB、権限、参照先URL生成および製品サーバは
このprofileに含めない。

== 契約version

machine-readableな検証registryはxref:../release-manifest.json[release manifest]である。Rustおよび
Workerの公開定数は、このmanifestとの一致をrelease gateで検査する。version変更では最初に
manifestを更新し、対応する実装、fixture、移行説明および以下の人向け一覧を同じ変更で同期する。

[cols="2,1,3",options="header"]
|===
|契約 |現在値 |更新する変更

|Core profile
|`CORE_PROFILE_VERSION = 2`
|構文認識、AST意味、置換、既定診断またはstrict・permissive動作の非互換変更

|Rust core API
|`CORE_API_VERSION = 8`
|公開Rust型、関数、エラーまたは意味契約の非互換変更

|HTML
|`HTML_CONTRACT_VERSION = 2`
|要素、属性、class、ID、URL処理または出力DOM意味の非互換変更

|Projection
|`PROJECTION_CONTRACT_VERSION = 2`
|検索テキスト、参照edge、通常リンクまたは安定JSON表現の非互換変更

|Conformance snapshot
|`CONFORMANCE_CONTRACT_VERSION = 4`
|CST・AST比較表現または比較対象productの非互換変更

|WASM API
|`WASM_API_VERSION = 2`
|WASM要求、応答、エラーまたは所有権契約の非互換変更

|Worker protocol
|`1`
|UIとWorker間のmessage、世代管理またはキャンセル契約の非互換変更
|===

これらのversionは独立している。たとえばHTMLだけの変更でCore API versionを更新せず、
WASM schemaだけの変更でWorker protocol versionを更新しない。複数契約へ影響する変更は、
該当するすべてのversionを同じ変更で更新する。

現行`SyntaxProfile.version`は過去profile実装を選択せず、現行値以外を拒否する互換性確認として
機能する。Issue 057では、単一profileだけを実装する期間にこの値を要求ごとに入力させる必要性を
再評価し、選択不能なversionは解析結果metadataとrelease manifestで公開する方向へ簡素化する。
複数profileを同時提供する場合だけ、実際のdispatchを伴うselectorを再導入する。

== 必須構文

profile 2の必須構文はxref:syntax-support.adoc[対応構文表]で「対応」「部分対応」または
「回復対応」と記載した範囲である。規範的な認識規則と置換順序は
xref:grammar.adoc[Core 0.2規範文法]およびxref:substitutions.adoc[置換モデル]を正本とする。

必須範囲には段落、タイトル、セクション、汎用文書属性、明示アンカー、リテラル・ソース
ブロック、順序付き・順序なしリストとblock continuation、constrainedおよびunconstrainedな
強調・斜体・等幅、
属性参照、通常リンク、文書内・文書間・任意スキームxrefおよびLaTeX STEMを含む。
一般passthrough、include、条件分岐、画像、表、脚注およびraw HTMLは
必須範囲外である。

`SyntaxMode::Permissive`は認識した未対応領域を保持し、安全なテキストとして処理する。
`SyntaxMode::Strict`は`Unsupported`ブロックを含む文書を公開処理前に拒否する。どちらの
modeでも未知構文をraw HTMLとして通過させない。

== 既定処理上限

[cols="2,1",options="header"]
|===
|資源 |Profile 2既定値
|入力 |10 MiB
|出力 |50 MiB
|1行 |1 MiB
|リスト深さ |8
|インライン深さ |32
|数式 |1 MiB
|ブロック |100,000
|semantic node |1,000,000
|参照 |100,000
|属性 |1,000
|診断 |1,000
|===

ホストは明示的に小さい値を指定できる。同じprofileを名乗るruntimeは、同じ入力と同じ
`ParseOptions`に対して同じ上限判断を行わなければならない。
公開上限値はruntimeのword sizeに依存しない`u32`で表す。入力長やnode数を扱う内部の
`usize`との比較では、`u32`から損失なく変換し、切り詰めるcastを行わない。

== crateのSemVer

すべてのworkspace crateは同じSemVerを使用する。`0.y.z`期間は次を適用する。

* patchは不具合修正に限定し、公開契約を意図的に破壊しない。
* minorは機能追加または破壊的変更を含められるが、影響する上記契約version、fixture、
  移行説明を同時に更新する。
* `1.0.0`以降はRust公開APIの破壊的変更をmajor、後方互換な機能追加をminor、修正をpatchとする。
* 診断codeの削除・意味変更、既定severityの意味変更および自動修正Editの意味変更は
  profile上の互換性変更として扱う。message文言だけは安定APIとしない。

crate SemVerだけからHTMLやprojectionの再構築要否を推測せず、個別契約versionを比較する。

== 再構築条件

[cols="2,3",options="header"]
|===
|変更された契約 |ホストが再構築する成果物

|Core profile、Core APIまたは解析option
|保存済み解析cache、診断、HTML、検索projection、参照projectionおよびシンボル

|HTML contractまたはHTML policy
|保存済みHTML・プレビュー。解析cacheとprojectionは変更がなければ再利用できる

|Projection contract
|検索索引、通常リンク索引、参照edgeおよび逆参照索引

|Resolver結果、対象文書versionまたは権限
|解決済み参照edge、逆参照索引および参照を含むHTML。元文書の構文解析は再利用できる

|診断設定だけ
|診断。CST・AST・HTML・projectionは設定が意味処理へ影響しない限り再利用できる

|Formatter設定だけ
|保存済み整形結果。解析・HTML・projectionの再構築理由にはならない
|===

cache keyと採用判定の詳細はxref:server-execution.adoc[ホスト実行・キャッシュ契約]に従う。
