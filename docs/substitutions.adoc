= 文字の置換とインライン書式

属性の展開、強調、リンクなどをどの順で読み取るかを説明します。日常的な書き方は
xref:syntax-support.adoc[対応しているAsciiDocの書式]を参照してください。

置換は出力バックエンドの前に、意味コンテキストごとの順序付きpipelineとして実行する。
HTMLその他のバックエンドは元ソースを再解析したり、属性を再展開したりしてはならない。

== 規範的な段階

[cols="1,3",options="header"]
|===
|段階 |責務

|`SpecialCharacters`
|出力先で意味を持つ文字を識別する。実際のエスケープはバックエンドが行う。

|`Quotes`
|強調、斜体、等幅、highlight、subscript、superscriptおよびtypographic quoteを意味ノードへ変換する。

|`Attributes`
|文書属性を再帰展開する。未定義、unset、escape、循環、深さ上限および出力byte上限を判定する。

|`Replacements`
|`(C)`、`(R)`、`(TM)`、省略記号、矢印および語中apostropheを置換する。

|`Macros`
|link、xref、anchor、STEMその他の対応済みmacroを意味ノードへ変換する。

|`PostReplacements`
|改行等、inline構造の確定後にのみ適用できる置換を表す。
|===

段階の列挙と順序の正本は`SubstitutionStep`である。新しい置換を追加するときは既存段階へ
所属させるか、新しい段階を型と本表へ同時に追加する。

== コンテキスト

[cols="1,4",options="header"]
|===
|コンテキスト |適用順

|`Normal`
|`SpecialCharacters` → `Quotes` → `Attributes` → `Replacements` → `Macros` → `PostReplacements`

|`Header`
|`SpecialCharacters` → `Attributes`

|`Verbatim`
|`SpecialCharacters`のみ。内容は不透明に保持する。

|`Pass`
|置換しない。passthroughはraw HTML権限を意味せず、通常のバックエンドでは安全にエスケープする。

|`None`
|置換しない。
|===

`SubstitutionContext::steps()`を実行時の正本とし、この表との一対一対応をテストする。

== 意味コンテキスト別の規範表

[cols="2,2,2,2",options="header"]
|===
|領域 |コンテキスト／解析 |属性評価 |出力責任

|段落、通常見出し、リスト項目
|`Normal`
|再帰展開
|テキストをバックエンドでエスケープ

|文書タイトル、著者、改訂
|`Header`
|再帰展開
|メタデータと表示文字列を各出力文脈でエスケープ

|インライン等幅、literal／source本体
|`Verbatim`
|なし
|内容を不透明なテキストとしてエスケープ

|inline／block passthrough
|`Pass`
|なし
|raw出力せずエスケープ。raw HTMLは別の明示的な権限契約がない限り禁止

|リンク・xrefの表示文字列
|`Normal`
|再帰展開
|表示文字列をエスケープ

|リンクターゲット
|inline解析なし
|再帰展開後の同じ値をLintとrendererがURL policyへ渡す
|許可済みURLだけを属性値としてエスケープ

|xrefターゲット
|inline解析なし
|コアでは展開しない
|host解決後のhrefをURL policyで検査してエスケープ

|標準macroのtarget
|inline解析なし
|再帰展開後の値をresource queryとURL policyへ渡す
|resource URLとIDを属性文脈でエスケープ

|標準macroの属性list
|macro kindごとの表示値
|現段階では属性内を再解析しない
|代替文字列をテキスト文脈でエスケープし、寸法は10進整数だけを出力

|通常表cell
|`Normal`
|再帰展開
|cell文字列をエスケープ

|AsciiDoc style cell
|子blockのcontent model
|子blockごとの置換context
|同じblock parserで構築した意味nodeを走査し、cell全体をinlineとして再解釈しない

|literal／verse表cell
|`Verbatim`
|なし
|`pre`内容としてエスケープ

|文書属性値
|inline解析なし
|参照時に再帰展開
|利用側コンテキストが安全化

|数式、`Unsupported`
|`None`
|なし
|不透明なテキストとしてエスケープ
|===

semantic ASTでは`Text`、`Literal`、`Styled`、`AttributeReference`、`Link`、
`Reference`、`Passthrough`および`Formula`がこの区別を保持する。段落は行ごとではなく、
改行を含む一つのinline列を所有する。

== 属性評価

属性参照は最終的な文書属性snapshotに対して再帰評価する。`\{name}`は参照を抑止し、
unset属性は未定義として扱う。評価中の同名属性再訪は循環、設定された最大深さの超過は
`DepthLimitExceeded`、生成byte数の超過は`SizeLimitExceeded`とする。失敗時は入力表記を
保持し、安定した診断を生成する。rendererが独自に再試行してはならない。

== エスケープとpassthrough

normal contextでは、対応済み開始記号の直前にあるバックスラッシュ一つが置換を抑止する。
verbatim、pass、noneではバックスラッシュも内容の一部である。passthroughの意味は
「AsciiDoc置換を行わない」であり、「任意HTMLを信頼して出力する」ではない。

fixture `fixtures/substitutions/pipeline.adoc`と、名前に`substitutions`を含むテストが、
context、段階順、循環、escapeおよび資源上限を固定する。
