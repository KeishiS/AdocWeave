= GitHub Releases配布契約

この文書は、AdocWeaveのCLI、Language Serverおよびbrowser packageを配布する際の規範的な入口を定義する。
machine-readableな正本はxref:../release/distribution-plan.json[`release/distribution-plan.json`]、配布manifestの
schemaはxref:../release/adocweave-dist-manifest.schema.json[`adocweave-dist-manifest.schema.json`]である。

== 配布境界

配布先はリポジトリ所有者自身のGitHubリポジトリにあるGitHub Releasesだけである。crates.io、npm、
Homebrew、WinGet、Zed Extension Registryその他の公的registryへ登録しない。Cargo workspaceとZed拡張は
`publish = false`、browser packageは`private = true`を維持し、release gateで検査する。

GitHubリポジトリはpublicでもprivateでもよい。private repositoryのassetを取得するCLI、Zed拡張または
frontend buildは、利用者が明示的に与えたGitHub認証を使用する。credentialをarchive、manifest、log、
source codeまたは配布例へ埋め込んではならない。

== Release train

workspace、CLI、LSP、WASM、browser packageおよびZed拡張は一つのpackage versionを共有する。受理するtagは
stableの`vX.Y.Z`とrelease candidateの`vX.Y.Z-rc.N`（`N >= 1`）だけである。tagとversionが一致しない場合は
build開始前に拒否する。

初回公開baselineでは、Core Profile、Core API、HTML、projection、conformanceおよびWASM APIの6契約を
すべて`1`とする。この一括resetは公開前に一度だけ行った操作であり、以後は各契約を独立に単調増加させる。

== Targetとarchive

初回native targetは次の2種類だけである。

* `aarch64-unknown-linux-musl`
* `x86_64-unknown-linux-musl`

GitHub Actionsでは、前者をnative ARM64の`ubuntu-24.04-arm`、後者をnative x86-64の`ubuntu-24.04`へ
割り当てる。plan、global artifact、host等のtarget非依存jobも`ubuntu-24.04`を使用する。target tripleは
生成物の実行環境、runner labelはbuild hostを表すため、両者を同一概念として扱わない。

CLIとLSPは`dist 0.32.0`が生成する`adocweave-cli-{target}.tar.xz`および
`adocweave-lsp-{target}.tar.xz`を改名せずに公開する。各native archiveは対応する実行ファイル、
`README.adoc`、`LICENSE-MIT`、`LICENSE-APACHE`および`THIRD_PARTY_NOTICES.adoc`を含む。詳細は
xref:native-release.adoc[Native CLI・Language Server配布契約]で定義する。

browser packageは`adocweave-browser-{version}.tar.xz`であり、`dist`のextra artifactとして同じreleaseへ
一度だけ統合する。内容と最小例はxref:../web-worker/README.adoc[`web-worker/README.adoc`]で定義する。
GitHubが自動生成するsource archiveはbinary配布契約に含めない。

== Distribution manifest

`adocweave-dist-manifest.json`はpackage version、source commit、6契約versionおよび5 archiveのmetadataを
保持する。各asset entryは名前、kind、target、archive形式、byte数、SHA-256および実行ファイル名を持つ。
path separatorは`/`、hash対象はuploadするarchiveのbyte列そのものとする。assetは名前の昇順、JSON objectの
keyは再帰的な辞書順とし、UTF-8、LF、末尾改行1個のpretty JSONとしてcanonicalizeする。

xref:../release/adocweave-dist-manifest.fixture.json[fixture]はschema、sort順およびcanonical JSONを固定する。
配布metadata名は`adocweave-dist-manifest.json`、`adocweave.spdx.json`および`sha256.sum`に固定する。
provenanceはarchiveへcredentialや署名fileを埋め込まず、GitHub Artifact Attestationsとして提供する。
実archiveからこれらを生成・検証する処理はIssue 085で実装する。

== 検証

`cargo make release-contract`はrelease train、tag、target、asset計画、非公開registry設定、schemaおよびfixtureを
検査する。`cargo make dist-plan`は固定した`dist 0.32.0`のJSON planを生成し、package、version、2 target、
archive名、実行ファイル、license、READMEおよびbrowser extra artifactが配布計画と一致することを検査する。
`cargo make release-gate`は両方を含む。
