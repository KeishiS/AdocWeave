= アンカーと見出しID

== 明示アンカー

次の二形式をブロックアンカーとして扱う。

[source,asciidoc]
----
[[stable-id]]
== 見出し

[#paragraph-id]
段落
----

`[[id,表示名]]` では参照時の既定表示名も指定できる。アンカー行は直後の空行でない
ブロックへ関連付ける。対象ブロックがない場合、または同じブロックへ複数のアンカーを
付けた場合は `invalid-anchor` とする。

IDを空にできない。空白、制御文字、`[`、`]`、`<`、`>`、`,` および `#` を
使用できない。それ以外のUnicode文字、結合文字および絵文字は使用できる。

明示IDはUnicode正規化も大小文字変換も行わず、UTF-8の元表記を意味上のIDとして
使用する。したがって、合成済み文字と結合文字列、`Case` と `case` は別IDである。
CSTは区切り記号を含む行全体を元のバイト列のまま保持する。

== 自動見出しID

明示アンカーがない見出しには次の手順でIDを生成する。

. `_` から開始する。
. Unicode英数字をUnicode小文字へ変換して残す。
. その他の一連の文字を `_` 一つへ置換する。
. 英数字が残らない場合は `_section` とする。
. 既存の明示IDまたは先行する自動IDと衝突する場合は `_2`、`_3` のように
  最小の連番を付ける。

明示アンカーは自動IDより優先する。HTMLの `id` と参照索引のIDは同じ生成結果を
使用し、レンダラーが別にIDを生成してはならない。

== 参照先索引

`reference_targets()` は文書タイトル、セクションおよびその他の明示アンカーを共通の
`ReferenceTarget` として返す。各要素は種別、ID、表示名、ID範囲および対象ブロック
範囲を持つ。見出しに明示IDが付いても、種別は文書タイトルまたはセクションのまま
保持する。同一入力からの順序と内容は決定的である。

重複IDは `duplicate-anchor`、不正または対象のない明示アンカーは
`invalid-anchor` として位置付きで診断する。
