= アーキテクチャ

AdocWeaveはソース認識、意味解釈、エディターサービス、出力バックエンドを分離する。

[source,text]
----
ソーステキスト
  -> 損失のないソース行とCST
  -> 出力形式に依存しないsemantic AST
       -> 文書サービス、Lint、Formatter
       -> HTMLバックエンド
       -> 将来の出力バックエンド

CLIとLanguage ServerはコアAPIのアダプターである。
----

semantic ASTはHTMLではなく言語上の意味を表現する。インライン内容では通常テキスト、
再解釈しないリテラル内容、スタイル付き子要素を区別する。ブロックノードはソース範囲と
意味値を保持するが、タグ、class、レンダリングオプションを保持しない。HTMLモジュールは
ASTへ依存するが、Parser、AST、Lint、Formatter、CLI、Language ServerはHTMLモジュールへ
依存しない。

コアはソースと設定を明示的に受け取り、ファイル、ネットワーク、環境変数、時刻、
外部コマンドへアクセスしない。これらの副作用はホストアダプターが所有する。
