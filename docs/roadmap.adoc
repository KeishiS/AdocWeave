== プリプロセッサdirectiveと外部資源

`include::`はblock macroではなく、文書構造を解析する前に動作するpreprocessor directiveである。
`ifdef`、`ifndef`および`ifeval`も同じ層で入力行を選択する。ファイルやURLへ暗黙にアクセスしない
コア方針を維持するため、実装時は次の境界を設ける。

* コアはdirective構文、target、属性およびsource rangeを認識する。
* ホストがresourceを取得し、不変snapshot、base URI、safe modeおよび許可schemeを指定する。
* コアが最大深さ、include件数、総byte数、展開nodeおよび循環を制限する。
* 展開後sourceと元sourceの位置対応を保持し、診断とLSP navigationをinclude元へ投影できる。
* URL includeはhostが明示的に取得したsnapshotだけを参照し、通常の解析APIはI/Oを行わない。

実装契約はxref:preprocessor.adoc[プリプロセッサとresource snapshot]に記載する。

参照基準はhttps://docs.asciidoctor.org/asciidoc/latest/directives/include/[Includes]および
https://docs.asciidoctor.org/asciidoc/latest/directives/conditionals/[Conditionals]とする。

GitHub ReleasesをCLI、Language Serverおよびbrowser向けリアルタイムHTML変換libraryの最初の配布先とする。
このlibraryは`adocweave-wasm`とWeb Worker adapterを一つのarchiveにし、frontendからの高頻度なAsciiDoc

== Webホスト向けの安全な適用境界

Issue 095から101では、任意schemeのxref、通常リンク、source block、STEMおよびresourceへ利用側profileを
適用するための汎用境界を整備した。URLは文書入力、解決済み参照および解決済みresourceの由来を型で区別し、
Resolver由来だけroot-relative URLを許可できる。WASMはnativeと同じRenderPolicyをHTMLへ適用する。

外部HTTP(S)リンクの固定`target`／`rel`、source言語allowlist、数式言語policy、source block・数式projection、
未解決参照の表示policy、fallback notice、画像・media・include capabilityを公開設定として追加した。
共通conformance fixtureは同一source revision、profileおよびRenderInputsからnativeとWASMが同じAST、診断、HTML、
projectionを生成することを確認する。

AdocWeaveは`note`、UUID、DB、ACL、Base URL、KaTeX、syntax highlighterおよびDOMを解釈しない。これらは
xref:web-host-adapter.adoc[Webホストadapter契約]に従う利用側adapterが所有する。

RC.3の受け入れ後、xref:../issues/107-resolved-reference-display-text.adoc[Issue 107]で、明示labelのないxrefへ
hostが解決した公開可能なタイトルをRenderInputsとして渡す汎用契約を追加した。明示label優先、
プレーンテキストescape、failure時の非漏洩をnative／WASM共通契約として固定し、`note`等のscheme固有意味はhostに残した。

== ドキュメント保守規則

リポジトリのドキュメントはAsciiDoc（`.adoc`）で記述し、`.md`ファイルを受け入れない。
構文を変更するたびに次を更新する。

* xref:syntax-support.adoc[対応表]
* xref:compatibility.adoc[外部仕様との互換性評価]
* xref:grammar.adoc[規範文法]
* 該当する場合はxref:substitutions.adoc[置換モデル]
* 対応するfixtureとテスト

`docs-check`は`.md`ファイルの混入だけを検査する。`adoc-check`は追跡対象の全`.adoc`を
決定的なpath順でAdocWeave自身へ渡し、寛容modeの診断をpath付きで報告する。解析失敗、
入力上限違反またはpanicによる異常終了はverifyとrelease gateを失敗させる。
