= Core 0.2 規範文法

この文書はAdocWeave 0.1コアプロファイルが受理する言語を定義する。「〜しなければ
ならない」という記述は規範的である。バイト範囲はUTF-8、ゼロ始まり、半開区間とする。

== 字句モデル

入力は正しいUTF-8でなければならない。行を次のように定義する。

[source,ebnf]
----
line        = content, [ line-ending ] ;
line-ending = LF | CRLF ;
----

単独のCRは内容である。lossless層は全バイトと元の改行を保持しなければならない。
別途指定しない限り、ブロックマーカーは列0でのみ認識する。

== 文書とブロック

[source,ebnf]
----
document       = { blank-line | heading | document-attribute | block-anchor
                 | source-block | literal-block
                 | unsupported | paragraph } ;
blank-line     = { " " | TAB } ;
paragraph      = paragraph-line, { paragraph-line } ;
paragraph-line = nonblank content not beginning another recognized block ;

heading        = heading-marker, [ " " ], heading-text ;
heading-marker = "=", { "=" } ;

literal-block  = "....", line-ending,
                 { literal-line },
                 "....", [ line-ending ] ;

source-block   = source-attribute, line-ending,
                 "----", line-ending,
                 { source-line },
                 "----", [ line-ending ] ;
source-attribute = "[source]"
                 | "[source,", horizontal-space*, language,
                   horizontal-space*, "]" ;
language       = nonempty text excluding "," and "]" ;
math-block     = ("[stem]" | "[latexmath]"), line-ending,
                 "++++", [ line-ending ], opaque-math,
                 "++++", [ line-ending ] ;
document-attribute = ":", attribute-name, [ "!" ], ":",
                     [ horizontal-space, attribute-value ]
                   | ":!", attribute-name, ":" ;
block-anchor   = "[[", anchor-id, [ ",", reference-text ], "]]"
               | "[#", anchor-id, "]" ;
----

文書内容より前に現れる最初の1個マーカーの見出しは文書タイトルである。2個から6個の
マーカーはセクションレベル1から5を表す。それ以外の位置または深さでも、問題情報を
付与した見出しノードとして保持し、利用側がソースを破棄せず回復できるようにする。

ソース属性の直後には正確な`----`行が必要であり、間に空行を許可しない。インデントした
属性と区切りは認識しない。文書タイトル直後の連続する文書属性だけを認識する。

リテラルおよびソースの内容は不透明である。開始区切りと等しい行がブロックを閉じる。
より長い行やインデントした区切り状の行は内容として扱う。

== インライン構文

[source,ebnf]
----
inline-sequence = { text | monospace | strong | emphasis
                  | attribute-reference | link | cross-reference
                  | inline-math } ;
monospace       = "`", literal-content, "`" ;
strong          = "*", inline-sequence, "*" ;
emphasis        = "_", inline-sequence, "_" ;
attribute-reference = "{", attribute-name, "}" ;
link             = URL, [ "[", inline-sequence, "]" ] ;
cross-reference  = "<<", anchor-id, [ ",", inline-sequence ], ">>"
                 | "xref:", reference-destination,
                   "[", inline-sequence, "]" ;
unordered-list-item = "*", { "*" }, (" " | "\t"), inline-sequence ;
ordered-list-item   = ".", { "." }, (" " | "\t"), inline-sequence ;
list-continuation   = "+", line-end,
                      (literal-block | source-block) ;
inline-math      = ("stem:[" | "latexmath:["), opaque-math, "]" ;
----

EBNFには次の区切り記号条件を追加する。

- 開始記号は、直後にUnicode scalarが存在して空白でも同じマーカーでもなく、直前の
  scalarが存在しないか英数字以外の場合に有効である。
- 終了記号は、直前にscalarが存在して空白でも同じマーカーでもなく、直後のscalarが
  存在しないか英数字以外の場合に有効である。
- 走査では最初の有効な開始記号を選び、その後の最初の同種の有効な終了記号を使う。
  同順位の場合はソース順に従う。
- 等幅内容は不透明である。強調と斜体の内容は再帰的に解析する。
- 数式内容は不透明であり、属性、書式、リンクその他のAsciiDoc置換を適用しない。
- URL、`link:`、`xref:`、`stem:`および`latexmath:`は、行頭または直前が空白、`(`、`[`、
  `{`、`<`、二重引用符もしくは単一引用符であるtoken境界からだけ認識する。escape用の
  バックスラッシュが直前にある場合は、その一つ前を境界判定に使う。短縮xrefの`<<`は
  このtoken境界を要求しない。
- 既定の再帰深さ上限は32である。上限に達したspanはテキストとし、
  `nesting-limit-exceeded`を報告する。

unconstrained形式はstrongの`**`、emphasisの`__`およびmonospaceの二重バッククォートを
境界に依存せず認識する。同種markerの連続runは左から2文字ずつpair候補へ分割し、最初の
開始候補に対する最初の同種候補を終了記号とする。空pairは通常テキストとして保持する。
対応済みの書式、属性参照、URL、macroおよびanchor開始記号の直前にある奇数個の
連続バックスラッシュは、その最後の一つで認識を抑止し、最後の一つだけを表示値から除く。
偶数個の場合は認識を抑止せず、バックスラッシュをそのまま表示する。escapeされた二重
markerでは最後のバックスラッシュと開始markerだけをliteral化し、対応する終了markerも
通常テキストとして残る。末尾の単独
バックスラッシュは通常テキストとして保持する。不透明context内ではescapeを解釈しない。

リンクとxrefのターゲットは不透明な字句として解析し、表示文字列だけを
`inline-sequence`として再帰解析する。認識優先順位は、ソース上で先に始まる
リンク・xrefまたは書式spanを優先し、同じ位置ではリンク・xrefを優先する。

== 回復規則

- 閉じていないインライン開始記号はテキストとして残して`unclosed-inline`を報告する。
  後続の安全なspanを失わないよう、そのマーカーの直後から走査を再開する。
- 閉じていないリンクまたはxrefの表示文字列はテキストとして残し、専用診断を付けて
  後続インライン走査を継続する。
- インライン走査は段落内の連続行を一つの列として扱い、LF／CRLFを内容とsource rangeに
  保持する。空行による段落境界または見出し境界は越えない。
- 閉じていない区切りブロックはファイル末尾までを内容とする。ただし列0で`=`から始まる
  見出し候補を検出した場合はそこで回復を開始する。候補は問題情報を含み得る見出しとして
  通常どおり解析し、開始区切りへ
  `unclosed-block`を報告する。
- 未対応構文は寛容モードで明示的な`Unsupported`ノードになる。厳格モードではこの
  ノードを含む文書を拒否する。
- 空入力、不完全入力、不正な見出しに対してもpanicせず、回復可能なCST/ASTを返す。

|===

|`AstBlock::Literal`

|`source-block`
|`AstBlock::Source`

|`block-anchor`
|`AstDocument::anchors`

|`AstBlock::Unsupported`

|通常インラインテキスト

|`Inline::Literal { kind: Monospace }`

|`Inline::Styled`

|`Inline::Link`

|`cross-reference`
|`Inline::Reference`

|`Inline::Formula`

|`math-block`
|`AstBlock::Math`
|===

`fixtures/grammar/ambiguous.adoc`を曖昧入力と回復の規範fixtureとする。名前に
`grammar`を含むテストが期待値を固定する。

== 将来の構文規則の追加

後続Issueで構文を追加する場合、同じ変更で次を定義しなければならない。

. 字句とEBNFまたは同等の認識規則
. 既存ブロック・インライン構文との認識優先順位
. CSTからsemantic ASTへの対応と、保持する元範囲
. 不完全入力の同期点、回復ノードおよび安定した診断コード
. 厳格・寛容モードでの扱い
. 曖昧入力、境界値および回復を固定するlossless fixture

新しい出力バックエンドの都合で文法またはsemantic ASTの意味を変更してはならない。
