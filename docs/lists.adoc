= リストとブロック継続

AdocWeaveは順序なし、順序付き、descriptionおよびcalloutの4種類を`ListKind`で区別する。
すべての項目はソース範囲、marker、本文、子listおよび0個以上のattached blockを保持する。

== 順序なし・順序付きlist

行頭の`*`と`.`をmarkerとし、同じ記号の反復数を深さとする。深さは既定で8までである。

[source,asciidoc]
----
* 項目
** 入れ子
. 順序付き項目
----

順序なし項目の本文先頭にある`[ ]`はunchecked、`[x]`、`[X]`または`[*]`はcheckedという
optionalな`ChecklistState`になる。状態markerは本文のinline列には含めない。

== Description list

`::`、`:::`、`::::`および`;;`を認識する。termとdescriptionは別々のinline列であり、
descriptionを持たない連続termは、次のdescriptionを共有する一つのterm集合になる。

[source,asciidoc]
----
Alias::
Term:: 説明
Parent:::
Child:::: 深い説明
----

== Callout

source block内の`<1>`と、callout listの`<1> 説明`は同じ正整数IDを持つ型付きノードになる。
`<.>`はlist内の出現順から決定的なIDを得る。escapeされたsource markerは対象外である。
HTMLはcallout番号と説明を安全な`ol`として出力し、source本文そのものは引き続きescapeする。

== Block continuation

単独の`+`は直前の項目へ次のblockをattachする。複数の`+`を順に使用でき、段落、list、
literal、sourceおよび標準delimited containerを混在させられる。attached blockは
`ListItem::continuations`に通常の`AstBlock`として格納するため、専用の縮小ASTを持たない。

== 置換、上限および診断

項目本文とdescription termにはnormal置換contextを適用する。literal／source等のattached blockは
各block自身の置換contextを維持する。深さは`max_list_depth`、全構築nodeは`max_nodes`、
継続数は`max_list_continuations`で制限する。

同じ深さでの不整合なmarker種別、深さの飛び越し、空項目、深さ上限超過およびtab区切りには
`inconsistent-list`診断を付ける。tab区切りには安全な空白置換Code Actionを提供する。

未実装なのは、indentだけによる継続、開始番号・番号形式、複雑なlist属性、および
callout markerの自動削除等のbackend固有表示である。
