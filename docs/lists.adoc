= リストとブロック継続

AdocWeaveは行頭の`*`による順序なしリストと、`.`による順序付きリストを扱う。
同じ記号の反復数が深さを表し、既定の最大深さは8である。

[source,asciidoc]
----
* 項目
** 入れ子
. 順序付き項目
----

項目本文には段落・見出しと同じconstrained／unconstrainedインライン置換を適用する。リスト項目は
semantic ASTで`ListBlock`と`ListItem`になり、種類、マーカー、区切り、本文、子リスト
およびソース範囲を保持する。

単独の`+`に続くリテラルブロックまたはソースブロックは、直前の項目へ継続ブロック
として関連付ける。不完全な継続は元ソースを保持する。

同じ深さでのマーカー種別変更、深さの飛び越し、空項目、上限超過およびタブ区切りには
`inconsistent-list`診断を付ける。タブ区切りだけは、意味を変えない空白置換を
Code Actionとして提供する。

現段階では、インデントによる入れ子、チェックリスト、説明リスト、項目内の通常段落、
任意ブロックへの継続、開始番号およびリスト属性は未対応である。
