= 整形

`adocweave format`は、対応している範囲だけを整えます。意味が分からない書式や、内容を変えるおそれがある箇所は
そのまま残します。元の文書を安全に保つため、ファイル全体を作り直すのではなく、必要な箇所だけを変更します。

== Syntax node別方針

構造化syntax treeから、整形してはならない最小の上位node範囲を導出する。literal、source、
数式、macro、inline span、error、unknownおよび未対応領域にはEditを生成しない。見出し、属性、
アンカーおよびリストも、専用の意味保存規則が実装されるまではバイト単位で保持する。

保護範囲外の段落と空行では、行末空白、連続空行、改行形式および最終改行を設定に従って
正規化できる。意味を理解していないnodeを暗黙に整形してはならない。

== 不変条件

* `format(format(x)) == format(x)`を満たす。
* 保護されたsyntax範囲の内容を変更しない。
* Editは重複せず、UTF-8境界に一致する。
* 整形後の文書を再解析でき、対応済み意味構造を保持する。

将来、見出しやリストを整形する場合は、そのsyntax kindを専用規則へ変更し、token単位のEdit、
意味同値性fixtureおよび不完全入力の保存テストを同時に追加する。未知・不完全領域のbyte保存を
優先し、semantic loweringを再実行しなければ安全性を判断できないEditは生成しない。
