= Formatter契約

Formatterは同じ`Analysis`に含まれるlossless CSTを使用し、ソースを再解析しない。
出力はソース全体の置換ではなく、重複しない`TextEdit`の集合として構築する。

== CSTノード別方針

`CstBlockKind::formatting_policy()`をノード別方針の単一の定義箇所とする。

NormalizeLineWhitespace::
段落と空行に適用する。行末空白、連続空行、改行形式および最終改行を設定に従って
正規化できる。

PreserveBytes::
見出し、文書属性、アンカー、リスト、リテラル、ソース、数式、未対応構文等に適用する。
現在はノード範囲内をバイト単位で保持する。

新しいCSTブロックを追加する場合、同じ変更で方針を明示しなければコンパイルが成功しない
網羅的matchを維持する。意味を理解していないノードを暗黙に整形してはならない。

== 不変条件

* `format(format(x)) == format(x)`を満たす。
* PreserveBytes領域の内容を変更しない。
* Editは範囲の重複や重複範囲を持たない。
* 整形後の文書を再解析でき、対応済み意味構造を保持する。

将来、見出しやリストを整形する場合は、そのCST kindを専用方針へ変更し、token単位の
Edit、意味同値性fixtureおよび不完全入力の保存テストを同時に追加する。
