= AdocWeave対応構文

この文書は現在のAdocWeaveコアが実装する動作を記録する。対応する部分集合を列挙する
製品上の対応表であり、AsciiDoc言語の完全な実装を主張するものではない。

実装済み構文の規範的規則はxref:grammar.adoc[Core 0.1規範文法]で定義する。
AsciiDoc言語全体との差異はxref:compatibility.adoc[AsciiDoc仕様との互換性]に記載する。

== 状態の定義

対応::
構文にsemantic AST表現があり、Parserと、該当するRenderer、診断、Formatter、
エディター機能を縦断してテストしている。

保持::
最終的な意味は未実装だが、入力を通常テキストまたは`Unsupported`ノードとして保持する。
HTML出力では保持したテキストをエスケープする。

未対応::
その構文についてAsciiDoc互換の認識または動作を保証しない。

== 対応済みブロック構文

[cols="2,1,3",options="header"]
|===
|構文 |状態 |現在の動作

|段落
|対応
|連続する非空行を段落にする。CSTでは元のソース範囲を利用できる。

|文書タイトル
|対応
|文書内容より前にある最初の1個マーカーの見出しを文書タイトルとし、既定で出力する。

|セクション見出し
|対応
|2個から6個の`=`マーカーをセクションレベル1から5とする。階層診断、文書シンボル、
hover情報、安定した生成IDを提供する。

|文書属性
|対応
|文書タイトル直後の定義・解除をlosslessに保持し、ノート用メタデータを型付きで検証する。

|明示アンカー
|対応
|`[[id]]`、`[[id,label]]`および`[#id]`を直後のブロックへ関連付け、共通参照先索引と
HTML IDに使用する。

|区切り付きリテラルブロック（`....`）
|対応
|本体を不透明として扱い、インライン置換を適用しない。

|ソースブロック（`[source,LANGUAGE]`と後続する`----`）
|部分対応
|言語と不透明な本体を意味表現へ変換する。その他のブロック属性とソース短縮記法には
対応しない。

|空行
|対応
|空白またはタブだけの行を空行とする。

|未知のブロック構文
|保持
|認識できた構文は寛容モードで`Unsupported`になる。厳格モードでは、このノードを
含む文書を全コア操作で拒否する。
|===

== 対応済みインライン構文

[cols="2,1,3",options="header"]
|===
|構文 |状態 |現在の動作

|強調（`*text*`）
|部分対応
|constrainedな単一マーカー形式だけを認識する。

|斜体（`_text_`）
|部分対応
|constrainedな単一マーカー形式だけを認識する。`snake_case`のような識別子中の
アンダースコアはテキストとして残す。

|等幅（pass:[`text`]）
|部分対応
|constrainedな単一マーカー形式だけを認識し、内容を不透明として扱う。

|強調と斜体の入れ子
|対応
|設定したインライン深さ上限まで、スタイル付き内容を再帰的に解析する。

|閉じていない書式
|回復対応
|開始記号をテキストとして残して診断を出し、後続の安全なspanを失わないよう走査を続ける。

|文書属性参照（`{name}`）
|対応
|文書タイトル直後の属性を一回だけ展開する。再帰展開は行わない。

|URL単体と`URL[label]`
|対応
|URLポリシーで許可したスキームだけをリンクにする。ターゲットと表示文字列の範囲を
分離して保持する。

|文書内xref（`<<id,label>>`、`xref:#id[label]`）
|対応
|共通`Reference`へ変換し、同じ文書のアンカー索引で解決する。

|文書間・スキーム付きxref
|構文対応
|Resolverなしで解析、整形、抽出できる。HTMLリンク化にはホストの安全な解決結果が必要。

|Unconstrained書式（`**text**`、`__text__`、二重バッククォート）
|未対応
|Core 0.1では二重マーカー形式に書式上の意味を与えない。

|バックスラッシュエスケープ
|部分対応
|通常リンク、xrefおよびxref短縮形式の直前では認識を抑止する。それ以外の書式、
属性参照、文字置換に対する一般的なエスケープは未対応。

|ハイライト、下付き、上付き、組版用引用符
|未対応
|対応する書式マーカー専用のsemantic nodeはない。

|インラインpassthrough
|未対応
|単一、二重、三重のプラス記号によるpassthroughを解釈しない。
|===

== 実装予定の構文

[cols="2,2,3",options="header"]
|===
|領域 |予定Issue |現在の扱い

|リストとブロック継続
|xref:../issues/032-lists-and-continuations.adoc[Issue 032]
|対応。`*`と`.`の反復による入れ子、およびリテラル・ソースブロックへの`+`継続を
扱う。詳細はxref:lists.adoc[リストとブロック継続]を参照。

|cross reference解決とノート参照
|xref:../issues/033-note-reference-macro.adoc[Issue 033]
|文書間xrefや`note:`のホスト側解決は行わない。

|STEMとLaTeX
|xref:../issues/034-latex-stem.adoc[Issue 034]
|数式マクロをテキストとして残す。
|===

== 例

.対応済みのconstrained書式
[source,asciidoc]
....
Use *strong*, _emphasis_, and `monospace`.
....

.未対応のunconstrained書式
[source,asciidoc]
....
some**strong**word
some__emphasis__word
some``monospace``word
....

.対応済みソースブロック部分集合
[source,asciidoc]
....
[source,rust]
----
fn main() {}
----
....
