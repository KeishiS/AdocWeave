= AdocWeave対応構文

この文書は現在のAdocWeaveコアが実装する動作を記録する。対応する部分集合を列挙する
製品上の対応表であり、AsciiDoc言語の完全な実装を主張するものではない。

この対応範囲は`CONTRACT_VERSION = 5`として公開する。version、SemVerおよび再構築条件は
xref:core-profile.adoc[AdocWeave Core Profile]で定義する。
実装済み構文の規範的規則はxref:grammar.adoc[Core 0.4規範文法]で定義する。
AsciiDoc言語全体との差異はxref:compatibility.adoc[AsciiDoc仕様との互換性]に記載する。

== 状態の定義

対応::
構文にsemantic AST表現があり、Parserと、該当するRenderer、診断、Formatter、
エディター機能を縦断してテストしている。

保持::
最終的な意味は未実装だが、入力を通常テキストまたは`Unsupported`ノードとして保持する。
HTML出力では保持したテキストをエスケープする。

未対応::
その構文についてAsciiDoc互換の認識または動作を保証しない。

== 対応済みブロック構文

[cols="2,1,3",options="header"]
|===
|構文 |状態 |現在の動作

|段落
|対応
|連続する非空行を段落にする。syntax treeでは元のソース範囲を利用できる。

|文書header
|対応
|連続した文書タイトル、著者行、改訂行および文書属性を認識する。最初の空行をheader終端とし、
著者名・email、改訂番号・日付・備考および`article`、`book`、`manpage`、`inline` doctypeを型付きで公開する。

|Preamble
|対応
|header終端から最初のsectionまたはbook partまでのblock列を、人工的なAST nodeを加えずprojectionとして公開する。

|文書タイトル
|対応
|文書内容より前にある最初の1個マーカーの見出しを文書タイトルとし、既定で出力する。

|セクション見出し
|対応
|2個から6個の`=`マーカーをセクションレベル1から5とする。階層診断、文書シンボル、
hover情報、安定した生成IDを提供する。

|Book partとdiscrete heading
|対応
|`book` doctypeにおける本文中のlevel-0見出しをpartとする。`[discrete]`または`[float]`付き見出しは
参照対象に含めるがsection階層およびOutlineから除外する。

|Indented literal paragraph
|対応
|空白またはtabで開始する連続行を専用literal paragraphとして保持し、inline置換を適用せず`pre`へ安全に出力する。

|Thematic breakとpage break
|対応
|単独行の`'''`と`<<<`を専用blockとして認識する。HTMLでは`hr`と`hr.page-break`へ変換する。

|文書属性
|対応
|文書タイトル直後の定義・解除をlosslessに保持する。属性値は汎用文字列であり、
アプリ固有メタデータの型検証はホストへ委譲する。

|共通ブロックメタデータ
|対応
|空行を挟まず直前に置いたblock title、`[[id]]`、`[#id]`、shorthand role／option、
positional／named element attributeをすべてのsemantic blockで共通表現として保持する。
未知のstyleや属性は意味を推測せず、HTML属性へ通過させない。

|明示アンカー
|対応
|`[[id]]`、`[[id,label]]`および`[#id]`を直後のブロックへ関連付け、共通参照先索引と
HTML IDに使用する。

|区切り付きリテラルブロック（`....`）
|対応
|本体を不透明として扱い、インライン置換を適用しない。

|汎用区切り付きblock
|対応
|comment（`////`）、example（`====`）、listing（`----`）、literal（`....`）、open（`--`）、
sidebar（`****`）、pass（`++++`）、quote（`____`）およびtable（`|===`、カスタムseparatorの`!===`等）を認識する。
内容はcompound／verbatim／raw／tableの型で区別し、同種blockは異なるdelimiter長で入れ子にできる。

|Admonition
|対応
|`NOTE`、`TIP`、`IMPORTANT`、`WARNING`、`CAUTION`のparagraph形式（`NOTE: text`）と、
example／open blockへ付けるblock形式（`[NOTE]`）を同じ型付きpresentationへ解決する。未知styleは通常blockとして保持する。

|Quoteとverse
|対応
|`[quote,Author,Source]`および`[verse,Author,Source]`を`____` blockへ付けると、引用種別、
attribution、citationを型付きpresentationへ解決する。quoteは`blockquote`、verseは改行を保持する`pre`として出力する。

|表
|対応
|PSV／CSV／DSV／TSV、空白・制御文字・`=`を除く1文字separator、quoted multiline cell、duplication、暗黙／明示列、
header／footer、column／row span、配置、cell styleおよびPSVのAsciiDoc block cellを型付きで扱う。
AsciiDoc cellは文書rootと同じblock sequence状態機械を用い、カスタムseparator delimiterによる入れ子表も扱う。
詳細はxref:tables.adoc[表の対応仕様]を参照する。

|ソースブロック（`[source,LANGUAGE]`、`[,LANGUAGE]`、`source-language`）
|対応
|標準形と短縮形を同じsource verbatim blockへ正規化する。`source-language`はlisting blockの既定言語となり、
明示した`[listing]`は既定値を抑止する。言語と不透明な本体はHTML、projection、LSPで共有する。

|空行
|対応
|空白またはタブだけの行を空行とする。

|未知のブロック構文
|保持
|認識できた構文は寛容モードで`Unsupported`になる。厳格モードでは、このノードを
含む文書を全コア操作で拒否する。

|順序なし・順序付きリスト
|対応
|`*`と`.`の反復による入れ子、およびリテラル・ソースブロックへの`+`継続を扱う。順序付きlistは
`start`、`reversed`および標準番号形式を型付きpresentationとして解決する。

|LaTeX STEMブロック
|対応
|`[stem]`または`[latexmath]`と`++++`による本体を、不透明な数式ノードとして保持する。
|===

== 対応済みインライン構文

[cols="2,1,3",options="header"]
|===
|構文 |状態 |現在の動作

|強調（`*text*`）
|対応
|constrainedな単一マーカーとunconstrainedな二重マーカーを認識する。

|斜体（`_text_`）
|対応
|constrainedな単一マーカーとunconstrainedな二重マーカーを認識する。
`snake_case`のような単一アンダースコアはテキストとして残す。

|等幅（pass:[`text`]）
|対応
|constrainedな単一マーカーとunconstrainedな二重マーカーを認識し、内容を不透明として扱う。

|強調と斜体の入れ子
|対応
|設定したインライン深さ上限まで、スタイル付き内容を再帰的に解析する。

|連続行をまたぐ書式とmacro label
|対応
|段落内のLF／CRLFを保持した一つのinline列として解析する。空行またはblock境界は越えない。

|Hard line break
|対応
|行末の空白と`+`を`HardBreak`として保持し、HTMLでは`br`へ変換する。通常の段落改行は従来どおり折り畳む。

|閉じていない書式
|回復対応
|開始記号をテキストとして残して診断を出し、後続の安全なspanを失わないよう走査を続ける。

|文書属性参照（`{name}`）
|対応
|文書タイトル直後の属性を一回だけ展開する。再帰展開は行わない。

|URL単体と`URL[label]`
|対応
|URLポリシーで許可したスキームだけをリンクにする。ターゲットと表示文字列の範囲を
分離して保持する。

|文書内xref（`<<id,label>>`、`xref:#id[label]`）
|対応
|共通`Reference`へ変換し、同じ文書のアンカー索引で解決する。

|文書間・スキーム付きxref
|構文対応
|Resolverなしで解析、整形、抽出できる。HTMLリンク化にはホストの安全な解決結果が必要。
空labelには解決成功と同じrevisionの任意のプレーンテキスト表示名を渡せる。

|インラインSTEM
|対応
|`stem:[...]`と`latexmath:[...]`を不透明なLaTeX数式ノードとして保持する。

|標準macroとmedia
|部分対応
|email／mailto、footnote、inline／bibliography anchor、index term、feature flag付きUI macro、
inline／block image、icon、audioおよびvideoを共通macro modelで扱う。footnoteの番号・再利用・back reference、
bibliography entry／参照edgeおよび階層index catalogに対応する。詳細は
xref:macros-and-resources.adoc[標準macroとresource境界]を参照する。

|Unconstrained書式（`**text**`、`__text__`、二重バッククォート）
|対応
|境界文字に依存せず認識する。空pairはテキストとして保持し、隣接pairは左から決定的に分割する。

|バックスラッシュエスケープ
|対応
|対応済み書式、属性参照、URL、xref、STEM macroおよびanchorの認識を抑止する。
不透明contextでは適用せず、未対応構文のescapeは対象外。

|ハイライト、下付き、上付き、組版用引用符
|対応
|`#highlight#`、`~subscript~`、`^superscript^`および曲線引用符のmarkerを型付きinline nodeとして保持し、
HTMLでは`mark`、`sub`、`sup`および対応するUnicode引用符へ変換する。

|インラインpassthrough
|対応
|単一、二重、三重のプラス記号と`pass:[...]`を不透明なinline nodeとして保持する。
HTMLへraw markupとして通過させず、テキストとしてescapeする。
|===

== 公式仕様に対する主な未実装領域

2026-07-21時点のAsciiDoc pre-specificationおよび公式言語文書との比較である。網羅的な実装順は
xref:roadmap.adoc[ロードマップ]を参照する。

[cols="2,1,4",options="header"]
|===
|領域 |状態 |不足している主な機能

|Block metadata
|対応
|block title、明示anchor、ID、role、option、positional／named属性を共通モデルへ保持する。
block titleは属性展開と通常inline構文を適用したbackend非依存のpresentationとして保持する。
block種別ごとの属性検証は今後のblock実装で追加する

|Delimited block
|部分対応
|標準delimiterとcontent model、異なるdelimiter長によるcompound入れ子に対応する。
compoundとAsciiDoc cellは文書rootと同じblock sequence状態機械で完全な対応済みblock列を解析する

|Document headerとsection
|対応
|著者、改訂、doctype、preamble、book part、appendix、discrete heading、`toc`、`toclevels`、`sectnums`、
manpageのname／section／purpose構造に対応する。詳細はxref:document-structure.adoc[文書構造契約]を参照する

|Paragraphとbreak
|対応
|通常段落、明示lead role、indented literal paragraph、hard line break、thematic breakおよびpage breakに対応する

|List
|対応
|順序なし、順序付き、description、checklist、callout、入れ子および複数のattached blockに対応。
順序付きlistは`start`、`reversed`、`arabic`、`decimal`、`loweralpha`、`upperalpha`、`lowerroman`、
`upperroman`および`lowergreek`を解決する。明示番号と`start`の整合性を診断する。任意custom counterおよび複雑なlist属性は未対応

|Table
|対応
|PSV／CSV／DSV／TSV、separator、quoted multiline cell、duplication、列指定、header／footer、
cell span、cell styleおよびPSVのAsciiDoc block cellに対応

|Inline macro
|対応
|URL、xref、email／mailto、image、footnote、anchor、bibliography anchor、kbd、btn、menuおよび
index termを意味nodeとして扱い、文書catalogへfootnote、bibliography、階層indexを投影する

|Media
|対応
|block／inline image、icon、audioおよびvideoをresource参照として公開する。取得と埋め込みはhostの責務

|置換pipeline
|対応
|6段階の順序、型付きcontext、文字置換、typographic quote、再帰属性展開、循環・資源上限、
安全なinline／block passthroughに対応。任意`subs`属性による段階の編集は未対応

|Preprocessor directive
|部分対応
|純粋なresource snapshot上でinclude、ifdef、ifndef、ifeval、escape、基本include属性およびsource mapに
対応。quote-awareなinclude属性値とmissing resourceだけを空展開する明示`optional` includeに対応。
wildcardおよびURI転送は未対応。詳細は
xref:preprocessor.adoc[プリプロセッサとresource snapshot]を参照

|Bibliographyとfootnote
|部分対応
|bibliography anchorとfootnote macroの構文node、名前付きfootnote再利用、出現順番号、back reference、
bibliography参照edgeおよび階層index catalogに対応する。`[bibliography]` section内の順序なしlistは
catalogのcitation／back referenceを出力する。bibliography styleのdoctype、document title／discrete heading、
multipart bookのsection scopeを検証する。citation styleの整形およびindex rangeは未対応

|Markdown互換構文
|未対応
|AsciiDoc標準の必須範囲ではなく、Asciidoctorのoptional機能として後順位にする
|===

== 例

.対応済みのconstrained書式
[source,asciidoc]
....
Use *strong*, _emphasis_, and `monospace`.
....

.対応済みのunconstrained書式
[source,asciidoc]
....
some**strong**word
some__emphasis__word
some``monospace``word
....

.対応済みソースブロック部分集合
[source,asciidoc]
....
[source,rust]
----
fn main() {}
----
....
