= 公開している仕様と移行情報

このページは、AdocWeaveをライブラリやWebAssemblyとして使う人のための入口です。
公開API、JSONの入出力、HTMLの出力は、すべてリリース番号で対応関係を確認します。
更新時に何を変更すればよいかは、下の移行ガイドを参照してください。

作業の経緯はIssue、設計を選んだ理由はADR、今後の予定はロードマップに分けて記録しています。

== 移行ガイド

過去のリリースから更新する利用側アプリは、現在使っている版より後の移行ガイドを番号順にすべて確認してください。
たとえばv0.1.xからv0.6.0へ更新する場合は、xref:upgrade-0-2.adoc[v0.2.0]、
xref:upgrade-0-3.adoc[v0.3.0]、xref:upgrade-0-4.adoc[v0.4.0]および
xref:upgrade-0-5.adoc[v0.5.0]およびxref:upgrade-0-6.adoc[v0.6.0]への移行をこの順で読みます。各ガイドの必要なRust版と対象リリースの
release manifestを確認し、保存しているHTMLや検索用の情報を作り直してください。

* xref:upgrade-0-2.adoc[v0.2.0]：表現力のあるリスト、文献、目次の追加
* xref:upgrade-0-3.adoc[v0.3.0]：表の見出しと表示属性の追加
* xref:upgrade-0-4.adoc[v0.4.0]：注意書き、引用、詩のブロックの追加
* xref:upgrade-0-5.adoc[v0.5.0]：公開Rust APIの整理とスタイルシートの追加
* xref:upgrade-0-6.adoc[v0.6.0]：文書属性を記述した順番と位置の取得

== 文書の分類

このディレクトリは現行仕様を記述する。xref:adr/index.adoc[ADR]は採用済みの長期的な
設計判断、xref:roadmap.adoc[ロードマップ]は未実装の計画、xref:../issues/README.adoc[Issue]
は完了履歴と作業単位を記録する。Issue本文を現行仕様の正本として参照してはならない。
