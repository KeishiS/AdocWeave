= Heading compatibility notes

AsciiLoom intentionally defines heading IDs as part of its own HTML contract:

- IDs start with `_`.
- Unicode alphanumeric characters are preserved and lowercased where applicable.
- Runs of other characters become a single `_`.
- Empty results use `_section`.
- Repeated IDs receive `_2`, `_3`, and so on in document order.

This is semantic compatibility with Asciidoctor, not a byte-for-byte promise. The exact IDs in `basic.html` are the normative AsciiLoom output.
